  <action>
Create TG-93 as a sub-workflow that sends dispatch notifications to the owner when TG-05 detects a new job or approved quote.

Workflow structure:

```
executeWorkflowTrigger (receives: customer_name, customer_phone, address, service, event_type, scheduled_time)
  -> Build Dispatch SMS (Code node)
  -> Log to dispatch_log (Supabase POST — status: 'pending')
  -> Send Dispatch SMS to Owner (Twilio to OWNER_PHONE)
  -> Update dispatch_log SET dispatch_sms_sent_at = NOW()
  -> Wait 2 Hours (n8n Wait node)
  -> Check dispatch_log (Supabase GET — is status still 'pending'?)
  -> IF still pending:
      -> Send Reminder SMS to Owner
      -> Update dispatch_log SET reminder_sent_at = NOW()
  -> ELSE: Done (owner already responded via TG-92)
```

Build Dispatch SMS Code node (from RESEARCH.md):
```javascript
const job = $input.first().json;
const address = job.address || 'No address';
const mapsUrl = 'https://maps.google.com/?q=' + encodeURIComponent(address);
const eventLabel = job.event_type === 'quote_approved' ? 'QUOTE APPROVED' : 'NEW JOB';
const message = [
  `${eventLabel} - ${job.customer_name || 'Unknown Customer'}`,
  `Service: ${job.service || 'Unknown service'}`,
  `Address: ${address}`,
  job.scheduled_time ? `Scheduled: ${job.scheduled_time}` : null,
  `Maps: ${mapsUrl}`,
  '',
  'Reply 1 to confirm, 2 to flag issue'
].filter(Boolean).join('\n');
return [{ json: { message, maps_url: mapsUrl, owner_phone: 'OWNER_PHONE', customer_name: job.customer_name, address, service: job.service, event_type: job.event_type, scheduled_time: job.scheduled_time, customer_phone: job.customer_phone } }];
```

Log to dispatch_log (Supabase REST POST):
```
POST https://lwtmvzhwekgdxkaisfra.supabase.co/rest/v1/dispatch_log
Headers: apikey, Authorization: Bearer SERVICE_KEY, Content-Type: application/json, Prefer: return=representation
Body: {
  "event_type": "{{ $json.event_type }}",
  "customer_name": "{{ $json.customer_name }}",
  "customer_phone": "{{ $json.customer_phone }}",
  "address": "{{ $json.address }}",
  "service_type": "{{ $json.service }}",
  "scheduled_time": "{{ $json.scheduled_time }}",
  "maps_url": "{{ $json.maps_url }}",
  "status": "pending"
}
```
IMPORTANT: Use `Prefer: return=representation` to get back the row with `id` — needed for the 2-hour check.

Send Dispatch SMS (Twilio node):
- From: TWILIO_PHONE
- To: OWNER_PHONE (owner Quo line)
- Message: `{{ $('Build Dispatch SMS').item.json.message }}`
- Use actual Twilio credential ID/name from n8n

Wait node:
- 2 hours (n8n Wait node with `amount: 2, unit: 'hours'`)

Check dispatch_log (Supabase REST GET):
```
GET https://lwtmvzhwekgdxkaisfra.supabase.co/rest/v1/dispatch_log?id=eq.{dispatch_id}&select=status
```
Read the dispatch_id from the POST response earlier.

IF status is still 'pending':
- Send reminder SMS: "REMINDER: {eventLabel} - {customer_name} at {address} still needs confirmation. Reply 1 or 2."
- Update dispatch_log SET reminder_sent_at = NOW()

NOTE: Owner dispatch SMS bypasses sms_consent check (business-to-owner communication). No rate limiting on owner notifications.

Deploy via n8n API. Save JSON locally. Do NOT activate yet.
