2. **Detect Source** (Code node — identifies twilio_sms, web_form, or unknown):
```javascript
const body = $input.first().json.body || $input.first().json;
let source = 'unknown';
let event_type = 'unknown';
if (body.From && body.Body !== undefined && body.MessageSid) {
  source = 'twilio_sms';
  event_type = 'inbound_sms';
} else if (body.first_name || body.email) {
  source = 'web_form';
  event_type = 'lead_capture';
}
let smsIntent = null;
if (source === 'twilio_sms') {
  const from = (body.From || '').replace(/[^0-9+]/g, '');
  const text = (body.Body || '').trim().toLowerCase();
  const isOwner = from === process.env.TG_OWNER_PHONE;
  if (isOwner && ['1', 'yes', 'confirm'].includes(text)) smsIntent = 'dispatch_confirm';
  else if (isOwner && ['2', 'no', 'flag'].includes(text)) smsIntent = 'dispatch_flag';
  else if (['stop', 'unsubscribe', 'quit', 'cancel', 'end'].includes(text)) smsIntent = 'opt_out';
  else if (['start', 'subscribe'].includes(text)) smsIntent = 'opt_in';
  else if (text === 'help') smsIntent = 'help';
  else smsIntent = 'general';
}
return [{ json: { source, event_type, smsIntent, idempotency_key: body.MessageSid || null, from_phone: body.From || null, message_body: body.Body || null, raw_payload: body } }];
```

6. **Switch on source** — three outputs: twilio_sms, web_form, unknown (branch handlers added in Task 2)

Wire: Webhook -> Detect Source -> Check Idempotency -> IF duplicate (yes: respond 200, no: Log to webhook_events -> Switch on source)

2. **Switch on smsIntent** (5 branches):

    a. **dispatch_confirm** (reply "1"):
      -> Update dispatch_log SET owner_response='1', owner_responded_at=NOW(), status='confirmed' WHERE status='pending' ORDER BY created_at DESC LIMIT 1

    b. **dispatch_flag** (reply "2"):
      -> Update dispatch_log SET owner_response='2', owner_responded_at=NOW(), status='flagged' WHERE status='pending' ORDER BY created_at DESC LIMIT 1

    c. **opt_out**:
      -> Update sms_consent SET status='opted_out' for this phone
      -> Send opt-out confirmation SMS via Twilio

    d. **opt_in**:
      -> Update sms_consent SET status='opted_in' for this phone
      -> Send opt-in confirmation SMS via Twilio

    e. **DEFAULT** (help, general, quote_request, schedule_request, confirm, cancel):
      -> **CRITICAL: sms_consent check BEFORE routing to TG-76.** This implements the can_send_sms() consent gate from locked decisions. Query sms_consent table:
         `GET /rest/v1/sms_consent?phone=eq.{from_phone}&status=eq.opted_out&select=id&limit=1`
         If a row is returned, the customer has opted out — do NOT call TG-76. Log to webhook_events with processing_status='blocked_opted_out' and stop this branch.
      -> **Rate limit check:** Query sms_sends:
         `GET /rest/v1/sms_sends?to_phone=eq.{from_phone}&sent_at=gte.{today_start_iso}&select=id`
         Use `Prefer: count=exact` header and parse content-range. If count >= 3, skip and log processing_status='rate_limited'.
      -> If consent OK and rate limit OK: Execute TG-76 sub-workflow with { from, messageText, messageSid }
      -> TG-76 handles reply + forwarding to owner
