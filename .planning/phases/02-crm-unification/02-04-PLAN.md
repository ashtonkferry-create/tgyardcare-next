  <files>automation/n8n-workflows/TG-94-unified-sms-sender.json</files>
  <action>
Create TG-94 as a sub-workflow that ALL customer-facing SMS must route through. It enforces consent, rate limits, and logging.

Workflow structure:

```
executeWorkflowTrigger (receives: to_phone, message_body, workflow_name, message_type)
  -> Check Consent (HTTP Request: Supabase RPC can_send_sms)
     POST https://lwtmvzhwekgdxkaisfra.supabase.co/rest/v1/rpc/can_send_sms
     Body: { "phone_number": "{{ to_phone }}" }
     Headers: apikey + Authorization: Bearer SERVICE_KEY
  -> IF (can_send = false)
     -> Return { sent: false, reason: "no_consent" }
  -> Check Rate Limit (HTTP Request: Supabase REST GET)
     GET https://lwtmvzhwekgdxkaisfra.supabase.co/rest/v1/sms_sends
       ?to_phone=eq.{{ to_phone }}
       &sent_at=gte.{{ today_start_iso }}
       &select=id
     Headers: Prefer: count=exact
  -> Code Node: extract count from content-range header
     const range = $input.first().json.headers?.['content-range'] || '';
     const count = parseInt(range.split('/')[1] || '0');
     return [{ json: { count, rate_limited: count >= 3 } }];
  -> IF (count >= 3)
     -> Return { sent: false, reason: "rate_limited" }
  -> Send SMS (Twilio node)
     From: TWILIO_PHONE
     To: {{ to_phone }}
     Body: {{ message_body }}
  -> Log to sms_sends (HTTP Request: Supabase REST POST)
     POST https://lwtmvzhwekgdxkaisfra.supabase.co/rest/v1/sms_sends
     Body: {
       "to_phone": "{{ to_phone }}",
       "message_body": "{{ message_body }}",
       "workflow_name": "{{ workflow_name }}",
       "message_type": "{{ message_type }}",
       "status": "sent",
       "sent_at": "{{ $now.toISO() }}"
     }
  -> Return { sent: true, message_sid: {{ twilio_response.sid }} }
```

IMPORTANT NOTES:
1. Owner dispatch SMS (TG-93) NEVER goes through this sub-workflow. TG-93 sends directly via Twilio node to OWNER_PHONE.
2. The can_send_sms() RPC already exists (migration 027). It returns a boolean.
3. The sms_sends table already exists (migration 027). Has columns: to_phone, message_body, workflow_name, message_type, status, sent_at.
4. Rate limit is 3 messages per customer per day (24h rolling window from midnight).
5. Use HTTP Request node for Supabase operations (not native Supabase node -- it lacks RPC and count support per RESEARCH.md Pitfall 4).
6. For rate limit count: use the Prefer: count=exact header and parse the content-range response header.
7. On Twilio send failure, log to sms_sends with status='failed' and return { sent: false, reason: "twilio_error" }.

Deploy via n8n API. Save JSON locally. Activate immediately -- this is a sub-workflow with no trigger of its own.
