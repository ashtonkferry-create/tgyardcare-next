Update the "Parse Jobber Email" Code node (id: parse-jobber-email) jsCode to add these extractions:

1. **scheduled_date extraction**: Parse date/time from email body. Jobber scheduled emails contain patterns like "Scheduled for March 25, 2026" or "Date: 03/25/2026" or "on Tuesday, March 25". Extract into ISO format (YYYY-MM-DD). Try multiple regex patterns:
   - `/scheduled\s+(?:for\s+)?(\w+ \d{1,2},?\s*\d{4})/i`
   - `/date[:\s]+(\d{1,2}\/\d{1,2}\/\d{2,4})/i`
   - `/on\s+((?:Monday|Tuesday|Wednesday|Thursday|Friday|Saturday|Sunday),?\s+\w+ \d{1,2})/i`
   If found, parse to YYYY-MM-DD string. If not found, set to null.

2. **scheduled_time extraction**: Look for time patterns like "9:00 AM", "between 8am-12pm". Extract as string (preserve original format). Regex: `/(?:at|time[:\s])\s*(\d{1,2}:\d{2}\s*(?:AM|PM)?(?:\s*-\s*\d{1,2}:\d{2}\s*(?:AM|PM)?)?)/i`

3. **crew_name extraction**: Look for patterns like "Assigned to: John" or "Crew: Team A" or "Technician: Mike Smith". Regex: `/(?:assigned to|crew|technician)[:\s]+([A-Za-z]+(?: [A-Za-z]+)?)/i`

4. **job_created event type**: Add detection BEFORE job_scheduled check. Pattern: `/job (?:has been )?created|new job/i` in subject. This is distinct from new_request (which is a customer request, not a Jobber-created job).

5. Add all new fields to the return object: `scheduled_date`, `scheduled_time`, `crew_name`.

Also update the "Insert Email Event" HTTP Request node (id: insert-email-event) jsonBody to include:
