## Part A: Seasonal Content Banner Component
`src/components/SeasonalContentBanner.tsx` (client component)

Logic:
```typescript
function getCurrentSeason(): 'spring' | 'summer' | 'fall' | 'winter' {
  const month = new Date().getMonth() + 1; // 1-12
  if (month >= 3 && month <= 5) return 'spring';
  if (month >= 6 && month <= 8) return 'summer';
  if (month >= 9 && month <= 11) return 'fall';
  return 'winter';
}
```

Banner shows relevant seasonal content at top of blog index:
- Spring (Mar-May): "It's Spring in Madison — time for cleanup, aeration & fertilization" → links to spring-related posts

Design: seasonal colors matching existing season theming (spring=green, summer=amber, fall=orange, winter=blue). Animated slide-in from top. Dismiss-able (localStorage).

## Part B: email_subscribers Migration
```sql
CREATE TABLE IF NOT EXISTS email_subscribers (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  email text NOT NULL UNIQUE,
  name text,
  source text DEFAULT 'blog',
  subscribed_at timestamptz DEFAULT now(),
  status text DEFAULT 'active' CHECK (status IN ('active', 'unsubscribed'))
);
ALTER TABLE email_subscribers ENABLE ROW LEVEL SECURITY;
-- Public insert only (anyone can subscribe), service_role can read all
CREATE POLICY "public_insert_subscribers" ON email_subscribers FOR INSERT WITH CHECK (true);
```
