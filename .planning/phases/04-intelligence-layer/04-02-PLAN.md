<task type="auto">
  <name>Task 2: Build and deploy TG-105 A/B Test Router</name>
  <files>automation/n8n-workflows/TG-105-ab-test-router.json</files>
  <action>
Build TG-105 as a sub-workflow called by sending workflows (TG-83, TG-84, TG-85, TG-91) before choosing message content.

Workflow structure:
```
Execute Workflow Trigger (typeVersion: 1)
  Input: { test_id, channel, recipient_phone?, recipient_email? }

  -> HTTP Request "Fetch Active Test"
     GET https://lwtmvzhwekgdxkaisfra.supabase.co/rest/v1/ab_tests?id=eq.{{ test_id }}&status=in.(active,winner_declared)&select=*
     Headers: apikey, Authorization (SUPABASE_SECRET_KEY_PLACEHOLDER), Prefer: return=representation
     fullResponse: true

  -> IF Node "Test Exists?"
     condition: {{ $json.body.length > 0 }}

  -> (false branch) Code Node "Return Default"
     Return { variant_id: null, variant_title: 'default', variant_content: null, is_default: true }

  -> (true branch) IF Node "Winner Declared?"
     condition: {{ $json.body[0].status === 'winner_declared' && $json.body[0].winner_variant_id }}

  -> (winner branch) HTTP Request "Fetch Winner Variant"
     GET .../ab_test_variants?id=eq.{{ winner_variant_id }}&select=*
     Return the winner variant content

  -> (no winner branch) HTTP Request "Fetch All Variants"
     GET .../ab_test_variants?test_id=eq.{{ test_id }}&select=*

  -> Code Node "Weighted Random Selection"
     // Get variants array
     const variants = $json.body;
     // Weighted random: sum weights, pick random number, find variant
     const totalWeight = variants.reduce((sum, v) => sum + (v.weight || 50), 0);
     let random = Math.random() * totalWeight;
     let selected = variants[0];
     for (const v of variants) {
       random -= (v.weight || 50);
       if (random <= 0) { selected = v; break; }
     }
     return [{ json: { variant_id: selected.id, variant_title: selected.title, variant_content: selected.content || selected.title, is_default: false, test_id } }];

  -> HTTP Request "Log Send to ab_test_sends"
     POST .../ab_test_sends
     Body: {
       test_id: {{ test_id }},
       variant_id: {{ selected.variant_id }},
       channel: {{ channel }},
       recipient_phone: {{ recipient_phone || null }},
       recipient_email: {{ recipient_email || null }}
     }

  -> Code Node "Return Selected Variant"
     Return { variant_id, variant_title, variant_content, is_default: false }
```

CRITICAL IMPLEMENTATION NOTES:
1. executeWorkflowTrigger MUST use typeVersion 1
2. Use PLACEHOLDER strings for Supabase keys
3. fullResponse: true on all Supabase GET requests
4. The ab_test_variants table stores variant content in the `title` field (original schema) — use `content` field if available, fall back to `title`
5. When test doesn't exist or isn't active, return is_default: true so the calling workflow uses its hardcoded default message
6. The log to ab_test_sends happens AFTER selection but BEFORE the calling workflow sends — this is fire-and-forget (don't block on it)
7. Weight field may not exist on all variants — default to 50 (equal weight)

Deploy via n8n API. Save JSON locally. Set workflow to ACTIVE.
