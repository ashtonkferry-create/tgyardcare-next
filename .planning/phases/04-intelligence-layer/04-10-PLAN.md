**TG-122 A/B Test Seed Manager:**
A sub-workflow (callable manually or from other workflows) that creates or updates A/B test configurations.

```
Execute Workflow Trigger (typeVersion: 1)
  Input: {
    action: 'create' | 'update',
    test_id: string (for create: suggested ID like 'quote-followup-sms-v1'),
    test_name: string,
    channel: 'sms' | 'email',
    variant_a: { title: string, content: string, weight: number },
    variant_b: { title: string, content: string, weight: number },
    min_sends_per_variant: number (default 30),
    auto_winner: boolean (default true)
  }

  -> IF "Create or Update?"

  -> (create) HTTP Request "Create Test"
     POST .../ab_tests
     Body: { id: test_id, test_name, channel, status: 'active', min_sends_per_variant, auto_winner }

  -> (create) HTTP Request "Create Variant A"
     POST .../ab_test_variants
     Body: { id: test_id + '-a', test_id, title: variant_a.title, content: variant_a.content, weight: variant_a.weight || 50 }

  -> (create) HTTP Request "Create Variant B"
     POST .../ab_test_variants
     Body: { id: test_id + '-b', test_id, title: variant_b.title, content: variant_b.content, weight: variant_b.weight || 50 }

  -> (update) HTTP Request "Update Test/Variants"
     PATCH .../ab_tests?id=eq.{{ test_id }}
     (and variant updates as needed)

  -> Return { success: true, test_id, action }
```

**TG-124 Lead Score Recalibrator:**
```
Schedule Trigger (cron: 0 13 1 * * = 1st of month 8 AM CT — same day as monthly reports)

  -> HTTP Request "Fetch All Leads with Scores"
     GET .../leads?lead_score=not.is.null&select=id,lead_score,status,source,total_revenue,created_at
     fullResponse: true

  -> Code Node "Analyze Score-to-Outcome Correlation"
     // Group leads by score tier (0-20, 21-40, 41-60, 61-80, 81-100)
     // For each tier:
     // - booking_rate = (status IN ('booked','completed')) / total
     // - avg_revenue = sum(total_revenue) / count
     // - avg_time_to_book (if timestamps available)
     //
     // Compare actual booking rates to expected rates
     // If a low-score tier books at high rates, scoring is miscalibrated
     //
     // Generate recommendations:
     // - "Source 'referral' converts at 60% but gets avg score 45 — increase source weight"
     // - "Score tier 61-80 has same booking rate as 81-100 — threshold needs adjustment"

  -> HTTP Request "Store Recalibration Report"
     POST .../intelligence_reports
     Body: { report_type: 'ab_test_results', report_period_start: monthStart, report_data: { tier_analysis, recommendations } }
     // Using existing report_type — or extend CHECK to include 'lead_scoring'

  -> Execute Sub-workflow TG-113 (if significant miscalibration found)
     severity: 'info'
     alert_body: recalibration recommendations
```

NOTE: TG-124 is analysis + recommendation only. It does NOT automatically update lead scoring weights (that would be dangerous without human review). It sends recommendations via TG-113.

```

     GET .../ab_tests?status=eq.active&channel=in.(sms,email)&select=id
```
