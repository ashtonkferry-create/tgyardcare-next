Workflow structure:
```
Schedule Trigger (cron: 0 12 * * * = daily 7 AM CT / 12 PM UTC)

  -> Code Node "Calculate Sync Date"
     const yesterday = new Date(Date.now() - 24*60*60*1000).toISOString().split('T')[0];
     return [{ json: { syncDate: yesterday, customerId: '$vars.TG_GOOGLE_ADS_CUSTOMER_ID' } }];

  -> IF Node "Ads Configured?"
     condition: {{ $vars.TG_GOOGLE_ADS_CUSTOMER_ID !== undefined && $vars.TG_GOOGLE_ADS_CUSTOMER_ID !== '' }}

  -> (false branch) Code Node "Skip - Not Configured"
     Return { status: 'skipped', reason: 'Google Ads API not configured' }

  -> (true branch) Code Node "Build GAQL Query"
     const query = `SELECT campaign.id, campaign.name, ad_group.id, ad_group.name, metrics.impressions, metrics.clicks, metrics.conversions, metrics.cost_micros, metrics.conversions_value, metrics.ctr, metrics.average_cpc, metrics.search_impression_share FROM ad_group WHERE segments.date = '${syncDate}'`;
     return [{ json: { query, syncDate } }];

  -> HTTP Request "Fetch Google Ads Data"
     POST https://googleads.googleapis.com/v17/customers/{{ customerId }}/googleAds:searchStream
     Headers:
       Authorization: Bearer {{ $vars.TG_GOOGLE_ADS_ACCESS_TOKEN }}
       developer-token: {{ $vars.TG_GOOGLE_ADS_DEV_TOKEN }}
       Content-Type: application/json
     Body: { "query": "{{ query }}" }

  -> Code Node "Transform Ads Data"
     // Parse Google Ads response (searchStream returns array of result batches)
     // Transform each row to google_ads_daily schema:
     // - cost = cost_micros / 1000000
     // - roas = conversion_value / cost (if cost > 0)
     // - avg_cpc = cost / clicks (if clicks > 0)

  -> HTTP Request "Upsert google_ads_daily"
     POST .../google_ads_daily
     Headers: Prefer: resolution=merge-duplicates,return=minimal
     Body: transformed data array
```

2. Google Ads API uses GAQL (Google Ads Query Language) — NOT SQL
3. cost_micros must be divided by 1,000,000 to get dollar amounts
4. Developer token and access token stored as n8n variables (PLACEHOLDER in local JSON)
5. Google Ads API requires OAuth token refresh — add a token refresh Code node if using refresh_token approach
6. If the API call fails, catch the error and log it (don't crash the workflow)

<task type="auto">
  <name>Task 2: Build and deploy TG-106 A/B Test Auto-Winner</name>
  <files>automation/n8n-workflows/TG-106-ab-test-auto-winner.json</files>
  <action>
Build TG-106 as a daily cron that checks all active SMS/email A/B tests and declares winners when statistical criteria are met.

Workflow structure:
```
Schedule Trigger (cron: 0 2 * * * = daily 9 PM CT / 2 AM UTC next day)

  -> HTTP Request "Fetch Active SMS/Email Tests"
     GET .../ab_tests?status=eq.active&channel=in.(sms,email)&auto_winner=eq.true&select=*
     fullResponse: true

  -> IF Node "Has Active Tests?"
     condition: {{ $json.body.length > 0 }}

  -> (false) Code Node "No Tests"
     Return { status: 'no_active_tests' }

  -> (true) SplitInBatches "Process Each Test"

  -> HTTP Request "Get Variant Send Counts"
     GET .../ab_test_sends?test_id=eq.{{ $json.id }}&select=variant_id,converted
     fullResponse: true

  -> Code Node "Calculate Winner Eligibility"
     // Group sends by variant_id
     // For each variant: total_sends, conversions, conversion_rate
     // Check if both variants have >= min_sends_per_variant
     // If yes, check if one variant has >20% relative improvement:
     //   Math.abs(rate_a - rate_b) / Math.max(rate_a, rate_b) > 0.20
     // Return: { eligible: true/false, winner_variant_id, stats }
     const sends = $json.body;
     const test = $('Fetch Active SMS/Email Tests').first().json.body.find(t => t.id === testId);
     const minSends = test.min_sends_per_variant || 30;

     const grouped = {};
     for (const s of sends) {
       if (!grouped[s.variant_id]) grouped[s.variant_id] = { total: 0, converted: 0 };
       grouped[s.variant_id].total++;
       if (s.converted) grouped[s.variant_id].converted++;
     }

     const variants = Object.entries(grouped);
     if (variants.length < 2) return [{ json: { eligible: false, reason: 'less_than_2_variants' } }];

     const [v1, v2] = variants;
     if (v1[1].total < minSends || v2[1].total < minSends) {
       return [{ json: { eligible: false, reason: 'insufficient_sends', v1_sends: v1[1].total, v2_sends: v2[1].total, min_required: minSends } }];
     }

     const rate1 = v1[1].converted / v1[1].total;
     const rate2 = v2[1].converted / v2[1].total;
     const relativeImprovement = Math.abs(rate1 - rate2) / Math.max(rate1, rate2, 0.001);

     if (relativeImprovement > 0.20) {
       const winnerId = rate1 > rate2 ? v1[0] : v2[0];
       return [{ json: { eligible: true, winner_variant_id: winnerId, rate1, rate2, relativeImprovement } }];
     }

     return [{ json: { eligible: false, reason: 'no_significant_difference', rate1, rate2, relativeImprovement } }];

  -> IF Node "Winner Found?"
     condition: {{ $json.eligible === true }}

  -> (true) HTTP Request "Declare Winner"
     PATCH .../ab_tests?id=eq.{{ testId }}
     Headers: Prefer: return=minimal
     Body: {
       status: 'winner_declared',
       winner_variant_id: {{ winner_variant_id }},
       winner_declared_at: new Date().toISOString()
     }

  -> (true) HTTP Request "Log to Intelligence Reports"
     POST .../intelligence_reports
     Body: {
       report_type: 'ab_test_results',
       report_period_start: start_date,
       report_period_end: today,
       report_data: { test_id, winner_variant_id, stats },
       highlights: [{ text: 'A/B test winner declared for {{ test_name }}' }]
     }
```

CRITICAL NOTES:
1. Runs at 9 PM CT — end of day, after all sends have been processed
2. Only processes tests where channel IN ('sms', 'email') AND auto_winner = true
3. The 20% relative improvement threshold is intentionally conservative for small sample sizes
4. SplitInBatches handles multiple active tests
5. fullResponse: true on all GET requests
6. When declaring winner, update ab_tests status to 'winner_declared' — TG-105 will then always return this variant
7. Log to intelligence_reports for the weekly report to pick up

Deploy via n8n API. Save JSON locally. Set workflow to ACTIVE.
