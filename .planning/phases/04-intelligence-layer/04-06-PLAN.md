Build TG-118 as a sub-workflow that takes structured report data and returns formatted HTML email content.

Workflow structure:
```
Execute Workflow Trigger (typeVersion: 1)
  Input: {
    report_title: string,           // e.g., "Weekly Revenue Attribution"
    report_subtitle: string,        // e.g., "March 10 - March 16, 2026"
    sections: [                     // array of report sections
      {
        type: 'metrics',            // metric cards row
        items: [
          { label: 'Total Revenue', value: '$4,250', change: '+12%', direction: 'up' },
          { label: 'Leads', value: '23', change: '-5%', direction: 'down' },
          ...
        ]
      },
      {
        type: 'highlights',         // bullet point highlights
        title: 'Key Insights',
        items: ['Google organic drove 60% of revenue', 'Quote follow-up SMS has 34% conversion rate']
      },
      {
        type: 'table',              // data table
        title: 'Revenue by Source',
        headers: ['Source', 'Revenue', 'Leads', 'Cost/Lead'],
        rows: [['Google Organic', '$2,550', '14', '$0'], ['Google Ads', '$1,200', '6', '$45']]
      },
      {
        type: 'anomalies',          // anomaly alerts
        title: 'Anomalies Detected',
        items: [{ metric: 'Lead Volume', deviation: '+45%', direction: 'spike' }]
      },
      {
        type: 'actions',            // recommended actions
        title: 'Recommended Actions',
        items: ['Increase Google Ads budget by 20%', 'A/B test winning SMS copy deployed']
      }
    ]
  }

  -> Code Node "Build HTML Email"
     Build complete HTML email with inline CSS:

     - Max-width: 600px, centered
     - Background: #f5f5f5, card background: #ffffff
     - Font: system-ui, -apple-system, sans-serif (email-safe)
     - Header: dark green (#1a5d1a) bar with TotalGuard logo text + report title
     - Subtitle: date range in lighter text
     - Metric cards: 2-3 per row, large number, label below, green/red change badge
     - Highlights: green left-border, bullet points
     - Tables: striped rows, right-aligned numbers
     - Anomalies: orange/red left-border based on severity
     - Actions: numbered list with bold text
     - Footer: "Generated by TotalGuard Intelligence Layer" + timestamp
     - All CSS inline (email clients strip <style> tags)

  -> Code Node "Return HTML"
     Return { html: fullEmailHtml, plain_text: stripHtml(fullEmailHtml) }
```

CRITICAL NOTES:
1. executeWorkflowTrigger MUST use typeVersion 1
2. ALL CSS must be inline — email clients (Gmail, Outlook) strip <style> blocks
3. Use table-based layout for email compatibility (not flexbox/grid)
4. Include both html and plain_text versions (plain_text for fallback)
5. The sub-workflow does NOT send the email — it returns the HTML for the caller to pass to TG-95
6. Keep the HTML template simple but professional — Vance is the only reader
7. Handle empty sections gracefully (skip rendering if items array is empty)
8. Handle missing optional fields (change, direction may not always be provided for metrics)
9. Max 5 section types: metrics, highlights, table, anomalies, actions

The Code Node should be a single comprehensive function that handles all section types. Keep it under 150 lines.

Deploy via n8n API. Save JSON locally. Set workflow to ACTIVE.
