  <name>Task 1: Build and deploy TG-116 Weekly Ad Performance</name>
  <files>automation/n8n-workflows/TG-116-weekly-ad-performance.json</files>
  <action>
Build TG-116 as a Monday morning cron that reports on Google Ads performance.

Workflow structure:
```
Schedule Trigger (cron: 0 13 * * 1 = Monday 8 AM CT)

  -> Code Node "Calculate Week Range"
     Last Monday to Sunday

  -> HTTP Request "Fetch Week's Ad Data"
     GET .../google_ads_daily?metric_date=gte.{{ weekStart }}&metric_date=lte.{{ weekEnd }}&select=*
     fullResponse: true

  -> HTTP Request "Fetch Previous Week Ad Data"
     GET .../google_ads_daily?metric_date=gte.{{ prevWeekStart }}&metric_date=lte.{{ prevWeekEnd }}&select=campaign_name,cost,clicks,conversions,conversion_value
     fullResponse: true

  -> HTTP Request "Fetch Week's Alerts"
     GET .../google_ads_alerts?created_at=gte.{{ weekStart }}&select=*
     fullResponse: true

  -> IF "Has Ad Data?"

  -> (false) Code Node "No Ads Report"
     // Build simple "No Google Ads data this week" report
     // Still useful to send — reminds Vance if ads aren't running

  -> (true) Code Node "Build Ad Report"
     // Aggregate by campaign:
     // - Total spend, clicks, conversions, ROAS per campaign
     // - Cost per lead: total spend / conversions
     // - Cost per booked job: total spend / job_booked conversions (if tracked)
     // - Week-over-week comparison
     // - Search impression share (opportunity metric)
     // - Any alerts triggered this week
     //
     // Sections:
     // - metrics: [Total Spend, Total Conversions, Avg CPC, Overall ROAS]
     // - table: Campaign Performance (name, spend, clicks, conversions, ROAS)
     // - highlights: Best/worst campaigns, trending keywords
     // - anomalies: Budget alerts, paused campaigns this week
     // - actions: Pause underperformers, increase budget on winners

  -> Execute Sub-workflow TG-118 (build HTML)
  -> Execute Sub-workflow TG-95 (send email)
     subject: 'TG Weekly Ads Report — {{ weekStart }} to {{ weekEnd }}'

  -> HTTP Request "Store Report"
     POST .../intelligence_reports
     Body: { report_type: 'ad_performance', report_period_start: weekStart, ... }
```

CRITICAL NOTES:
1. Gracefully handles "no data" — sends brief report instead of erroring
2. ROAS = conversion_value / cost (return 0 if cost = 0)
3. Include alerts from google_ads_alerts that were triggered during the week
4. If ads aren't running at all, the report serves as a nudge
5. fullResponse: true on all GET requests

Deploy. Save JSON. Set ACTIVE.
