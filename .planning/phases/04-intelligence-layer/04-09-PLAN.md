Build TG-119 as a monthly cron that compares this month vs last month across all tracked KPIs.

Workflow structure:
```
Schedule Trigger (cron: 0 13 1 * * = 1st of month 8 AM CT)

  -> Code Node "Calculate Month Ranges"
     // This month = previous calendar month (report on the 1st of the new month)
     // Last month = 2 months ago
     const now = new Date();
     const thisMonthStart = new Date(now.getFullYear(), now.getMonth() - 1, 1).toISOString().split('T')[0];
     const thisMonthEnd = new Date(now.getFullYear(), now.getMonth(), 0).toISOString().split('T')[0];
     const lastMonthStart = new Date(now.getFullYear(), now.getMonth() - 2, 1).toISOString().split('T')[0];
     const lastMonthEnd = new Date(now.getFullYear(), now.getMonth() - 1, 0).toISOString().split('T')[0];

  -> HTTP Request "Fetch This Month Metrics"
     GET .../intelligence_metrics?metric_date=gte.{{ thisMonthStart }}&metric_date=lte.{{ thisMonthEnd }}&select=metric_name,metric_value,metric_date
     fullResponse: true

  -> HTTP Request "Fetch Last Month Metrics"
     GET .../intelligence_metrics?metric_date=gte.{{ lastMonthStart }}&metric_date=lte.{{ lastMonthEnd }}&select=metric_name,metric_value,metric_date
     fullResponse: true

  -> Code Node "Calculate Trends"
     // For each metric_name:
     // 1. Calculate monthly average (sum / days with data)
     // 2. Calculate month-over-month change %
     // 3. Identify biggest movers (top 3 up, top 3 down)
     // 4. Flag any sustained anomalies (metric consistently above/below average)
     //
     // Sections:
     // - metrics: [Avg Daily Revenue, Total Leads, Conversion Rate, Email Open Rate]
     // - table: All Metrics (metric, this_month_avg, last_month_avg, change%)
     // - highlights: "Month in Review" (3-5 key trends)
     // - actions: Trend-based recommendations

  -> Execute Sub-workflow TG-118 (build HTML)
     report_title: "Monthly Trend Analysis"
     report_subtitle: "{{ monthName }} 2026 vs {{ prevMonthName }} 2026"

  -> Execute Sub-workflow TG-95 (send email)
     subject: 'TG Monthly Trends — {{ monthName }} 2026'

  -> HTTP Request "Store Report"
     POST .../intelligence_reports
     Body: { report_type: 'monthly_trend', ... }
```

Deploy. Save JSON. Set ACTIVE.

**TG-120 Monthly Channel ROI:**
```
Schedule Trigger (cron: 0 13 1 * * = 1st of month 8 AM CT)

  -> Calculate previous month range

  -> HTTP Request "Fetch Month Revenue by Source"
     GET .../revenue_attribution?attribution_date=gte.{{ monthStart }}&attribution_date=lte.{{ monthEnd }}&select=lead_source,revenue
     fullResponse: true

  -> HTTP Request "Fetch Month Ad Spend"
     GET .../google_ads_daily?metric_date=gte.{{ monthStart }}&metric_date=lte.{{ monthEnd }}&select=cost,conversions,conversion_value
     fullResponse: true

  -> HTTP Request "Fetch Month Leads by Source"
     GET .../leads?created_at=gte.{{ monthStart }}&created_at=lte.{{ monthEnd }}&select=source,status
     fullResponse: true

  -> Code Node "Calculate Channel ROI"
     // For each channel:
     // - Revenue generated
     // - Cost (ad spend for paid, $0 for organic)
     // - ROI = (revenue - cost) / cost * 100 (for paid)
     // - Leads generated
     // - Cost per lead
     // - Conversion rate
     //
     // Channels: google_organic, google_ads, referral, direct, nextdoor, facebook, yard_sign
     //
     // Sections:
     // - metrics: [Total Revenue, Total Ad Spend, Blended ROAS, Best Channel]
     // - table: Channel ROI (channel, revenue, cost, ROI%, leads, cost/lead)
     // - highlights: Best/worst channels, ROI insights
     // - actions: Channel budget allocation recommendations

  -> Execute Sub-workflow TG-118 + TG-95
  -> Store to intelligence_reports { report_type: 'channel_roi' }  // Note: extend CHECK or use 'revenue_attribution' type
```
