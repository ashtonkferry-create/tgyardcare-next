  <action>
Build TG-110 as a daily cron that compares today's KPI snapshot to the 4-week rolling average and alerts on significant deviations.

Workflow structure:
```
Schedule Trigger (cron: 0 14 * * * = daily 9 AM CT / 2 PM UTC)

  -> Code Node "Calculate Date Windows"
     const today = new Date().toISOString().split('T')[0];
     const fourWeeksAgo = new Date(Date.now() - 28*24*60*60*1000).toISOString().split('T')[0];
     return [{ json: { today, fourWeeksAgo } }];

  -> HTTP Request "Fetch Today's Metrics"
     GET .../intelligence_metrics?metric_date=eq.{{ today }}&select=metric_name,metric_value
     fullResponse: true

  -> HTTP Request "Fetch 4-Week History"
     GET .../intelligence_metrics?metric_date=gte.{{ fourWeeksAgo }}&metric_date=lt.{{ today }}&select=metric_name,metric_value
     fullResponse: true

  -> Code Node "Detect Anomalies"
     // For each metric in today's data:
     // 1. Calculate 4-week average from history
     // 2. Calculate deviation: |today - avg| / avg
     // 3. If deviation > 0.25 (25%), flag as anomaly
     // 4. Determine direction: 'spike' or 'drop'
     // 5. Determine severity: >50% = 'critical', >25% = 'warning'

     const todayMetrics = $('Fetch Today\'s Metrics').first().json.body;
     const history = $('Fetch 4-Week History').first().json.body;

     // Group history by metric_name, calculate average
     const avgs = {};
     for (const h of history) {
       if (!avgs[h.metric_name]) avgs[h.metric_name] = { sum: 0, count: 0 };
       avgs[h.metric_name].sum += parseFloat(h.metric_value);
       avgs[h.metric_name].count++;
     }

     const anomalies = [];
     for (const m of todayMetrics) {
       const avg = avgs[m.metric_name];
       if (!avg || avg.count < 7) continue; // Need at least 1 week of history
       const avgValue = avg.sum / avg.count;
       if (avgValue === 0) continue; // Can't calculate deviation from zero
       const deviation = Math.abs(parseFloat(m.metric_value) - avgValue) / avgValue;
       if (deviation > 0.25) {
         anomalies.push({
           metric_name: m.metric_name,
           today_value: parseFloat(m.metric_value),
           avg_value: Math.round(avgValue * 100) / 100,
           deviation: Math.round(deviation * 100),
           direction: parseFloat(m.metric_value) > avgValue ? 'spike' : 'drop',
           severity: deviation > 0.50 ? 'critical' : 'warning'
         });
       }
     }
     return [{ json: { anomalies, count: anomalies.length } }];

  -> IF Node "Has Anomalies?"
     condition: {{ $json.count > 0 }}

  -> (true) Code Node "Build Alert Body"
     // Format anomalies into HTML table for email
     // and short summary for SMS

  -> (true) Execute Sub-workflow "Send Alert via TG-113"
     workflowId: {{ TG-113 ID from 04-02 }}
     Input: {
       alert_type: 'kpi_anomaly',
       alert_title: '{{ count }} KPI anomalies detected',
       alert_body: htmlTable,
       severity: anomalies.some(a => a.severity === 'critical') ? 'critical' : 'warning',
       source_workflow: 'TG-110',
       metadata: { anomalies }
     }
```

CRITICAL NOTES:
1. Runs at 9 AM CT — after TG-108 (KPI snapshot at 6 AM CT) has written today's data
2. Needs at least 7 days of history before alerting (avoid false positives on new metrics)
3. Skip metrics with avg = 0 (division by zero)
4. severity escalation: >50% deviation = critical (triggers SMS), 25-50% = warning (email only)
5. fullResponse: true on all GET requests
6. TG-113 workflow ID comes from 04-02-SUMMARY.md — use placeholder if not yet known

Deploy via n8n API. Save JSON locally. Set workflow to ACTIVE.

  -> (true) Code Node "Build Pause List"
