Workflow structure:
```
Schedule Trigger (cron: 0 10 * * * = daily 5 AM CT / 10 AM UTC)

  -> Code Node "Calculate Sync Window"
     // Sync yesterday's data
     const yesterday = new Date(Date.now() - 24*60*60*1000).toISOString().split('T')[0];
     const today = new Date().toISOString().split('T')[0];
     return [{ json: { start_date: yesterday, end_date: today } }];

  -> HTTP Request "Fetch Payment Events"
     GET .../jobber_email_events?event_type=eq.payment_received&created_at=gte.{{ start_date }}&created_at=lt.{{ end_date }}&select=*
     Headers: apikey, Authorization (SUPABASE_SECRET_KEY_PLACEHOLDER)
     fullResponse: true

  -> IF Node "Has Payments?"
     condition: {{ $json.body.length > 0 }}

  -> (true) Code Node "Extract Revenue Data"
     // For each payment event, extract:
     // - customer_name, customer_email, customer_phone from parsed_data
     // - amount from parsed_data.amount
     // - invoice_number if available
     // Return array of revenue records

  -> HTTP Request "Match Leads"
     // For each payment, find the matching lead by email or phone
     GET .../leads?or=(email.eq.{{ email }},phone.eq.{{ phone }})&select=id,source,utm_source,utm_medium
     fullResponse: true

  -> Code Node "Build Attribution Records"
     // For each matched payment+lead pair:
     // {
     //   lead_id: lead.id,
     //   revenue: payment.amount,
     //   lead_source: lead.source || 'unknown',
     //   attribution_date: payment.date,
     //   workflow_attribution: 'jobber_email_parser',
     //   service_type: payment.service_type || 'general'
     // }

  -> HTTP Request "Upsert Revenue Attribution"
     POST .../revenue_attribution
     Headers: Prefer: resolution=merge-duplicates,return=minimal
     Body: array of attribution records

  -> Code Node "Log Sync Result"
     // Return summary: payments_processed, revenue_total, leads_matched, leads_unmatched
```

<task type="auto">
  <name>Task 2: Build and deploy TG-108 KPI Daily Snapshot</name>
  <files>automation/n8n-workflows/TG-108-kpi-daily-snapshot.json</files>
  <action>
Build TG-108 as a daily cron that snapshots ALL key business metrics into the intelligence_metrics table. This is the data source for anomaly detection (TG-110) and trend analysis.

Workflow structure:
```
Schedule Trigger (cron: 0 11 * * * = daily 6 AM CT / 11 AM UTC)

  -> Code Node "Get Today's Date"
     const today = new Date().toISOString().split('T')[0];
     return [{ json: { metric_date: today } }];

  -> HTTP Request "Count Today's Leads"
     GET .../leads?select=id&created_at=gte.{{ today }}
     Headers: Prefer: count=exact
     fullResponse: true
     // Extract count from content-range header

  -> HTTP Request "Count Leads by Source"
     GET .../leads?select=source,id&created_at=gte.{{ today }}&source=not.is.null
     fullResponse: true

  -> HTTP Request "Get Revenue Today"
     GET .../revenue_attribution?select=revenue&attribution_date=eq.{{ today }}
     fullResponse: true

  -> HTTP Request "Get Conversion Rate"
     // leads with status 'booked' or 'completed' vs total leads (last 7 days for stability)
     GET .../leads?select=id,status&created_at=gte.{{ 7_days_ago }}
     fullResponse: true

  -> HTTP Request "Get SMS Response Rate"
     GET .../sms_sends?select=id,replied&sent_at=gte.{{ yesterday }}
     fullResponse: true

  -> HTTP Request "Get Email Open Rate"
     GET .../email_sends?select=id,opened_at&sent_at=gte.{{ yesterday }}
     fullResponse: true

  -> HTTP Request "Get Avg Job Value"
     GET .../revenue_attribution?select=revenue&attribution_date=gte.{{ 30_days_ago }}
     fullResponse: true

  -> Code Node "Calculate All Metrics"
     // Aggregate all fetched data into metrics array:
     const metrics = [
       { metric_name: 'leads_total', metric_value: leadCount },
       { metric_name: 'revenue_total', metric_value: revenueSum },
       { metric_name: 'avg_job_value', metric_value: avgRevenue },
       { metric_name: 'conversion_rate', metric_value: bookedLeads / totalLeads * 100 },
       { metric_name: 'sms_response_rate', metric_value: repliedSms / totalSms * 100 },
       { metric_name: 'email_open_rate', metric_value: openedEmails / totalEmails * 100 },
       { metric_name: 'quote_conversion_rate', metric_value: acceptedQuotes / totalQuotes * 100 },
     ];
     // Add per-source lead counts
     for (const source of ['google_organic', 'google_ads', 'referral', 'direct', 'nextdoor', 'facebook']) {
       metrics.push({ metric_name: `leads_by_source_${source}`, metric_value: countBySource[source] || 0 });
     }
     // Add metric_date to all
     return metrics.map(m => ({ json: { ...m, metric_date: today } }));

  -> HTTP Request "Upsert Intelligence Metrics"
     POST .../intelligence_metrics
     Headers: Prefer: resolution=merge-duplicates,return=minimal
     Body: {{ $json }} (array of metric objects)
```

CRITICAL NOTES:
1. Cron at 6 AM CT — runs AFTER TG-107 (revenue sync at 5 AM CT) so revenue data is fresh
2. Use Supabase count via Prefer: count=exact + content-range header for efficient counting
3. Some metrics use 7-day or 30-day windows for stability (conversion rate, avg job value)
4. Daily metrics use yesterday's data (SMS/email) or today's running count (leads)
5. The UNIQUE index on (metric_date, metric_name) enables upsert via merge-duplicates
6. If a metric query returns no data, use 0 (not null) — anomaly detection needs numeric values
7. fullResponse: true on ALL Supabase GET requests
8. Multiple HTTP Request nodes can run in parallel if n8n supports it — but sequential is fine

Deploy via n8n API. Save JSON locally. Set workflow to ACTIVE.
