  <action>
Build TG-114 as a Monday morning cron that reports revenue by source, by service, and cost per lead.

Workflow structure:
```
Schedule Trigger (cron: 0 13 * * 1 = Monday 8 AM CT / 1 PM UTC)

  -> Code Node "Calculate Week Range"
     // Last Monday to Sunday
     const today = new Date();
     const lastMonday = new Date(today);
     lastMonday.setDate(today.getDate() - 7);
     const lastSunday = new Date(today);
     lastSunday.setDate(today.getDate() - 1);
     return [{ json: { weekStart: lastMonday.toISOString().split('T')[0], weekEnd: lastSunday.toISOString().split('T')[0] } }];

  -> HTTP Request "Fetch Week's Revenue"
     GET .../revenue_attribution?attribution_date=gte.{{ weekStart }}&attribution_date=lte.{{ weekEnd }}&select=*
     fullResponse: true

  -> HTTP Request "Fetch Week's Leads"
     GET .../leads?created_at=gte.{{ weekStart }}&created_at=lte.{{ weekEnd }}&select=id,source,status
     fullResponse: true

  -> HTTP Request "Fetch Previous Week Revenue" (for comparison)
     GET .../revenue_attribution?attribution_date=gte.{{ prevWeekStart }}&attribution_date=lte.{{ prevWeekEnd }}&select=revenue,lead_source
     fullResponse: true

  -> Code Node "Build Report Data"
     // Calculate:
     // 1. Total revenue this week vs last week (% change)
     // 2. Revenue by source: google_organic, google_ads, referral, direct, etc.
     // 3. Revenue by service_type: lawn_care, fertilizer, hardscaping, etc.
     // 4. Cost per lead by source (if ad_spend data available from google_ads_daily)
     // 5. Lead-to-revenue conversion rate by source
     // 6. Top 3 highlights (biggest revenue source, fastest growing, highest conversion)
     //
     // Format into TG-118 sections structure:
     // - metrics: [Total Revenue, Leads, Avg Job Value, Conversion Rate]
     // - table: Revenue by Source
     // - table: Revenue by Service
     // - highlights: Key Insights
     // - actions: Recommended Actions

  -> Execute Sub-workflow "Build HTML via TG-118"
     workflowId: {{ TG-118 ID }}
     Input: { report_title, report_subtitle, sections }

  -> Execute Sub-workflow "Send via TG-95"
     workflowId: IUDLrQrAkcLFLsIC
     Input: {
       to_email: 'totalguardllc@gmail.com',
       subject: 'TG Weekly Revenue Report — {{ weekStart }} to {{ weekEnd }}',
       html_body: {{ html from TG-118 }},
       workflow_name: 'TG-114',
       email_type: 'report'
     }

  -> HTTP Request "Store Report"
     POST .../intelligence_reports
     Headers: Prefer: resolution=merge-duplicates,return=minimal
     Body: { report_type: 'revenue_attribution', report_period_start: weekStart, report_period_end: weekEnd, report_data, highlights }
```

CRITICAL NOTES:
1. fullResponse: true on all GET requests
2. Week-over-week comparison drives the change % on metric cards
3. If no revenue data exists for the week, still send report with "No revenue recorded" message
4. Store report in intelligence_reports for historical access
5. TG-118 ID comes from 04-06-SUMMARY.md

Deploy via n8n API. Save JSON locally. Set to ACTIVE.

Build TG-115 as a Monday morning cron that reports on all A/B test activity.

Workflow structure:
```
Schedule Trigger (cron: 0 13 * * 1 = Monday 8 AM CT / 1 PM UTC)

  -> HTTP Request "Fetch Active Tests"
     GET .../ab_tests?channel=in.(sms,email)&status=in.(active,winner_declared)&select=*
     fullResponse: true

  -> HTTP Request "Fetch This Week's Sends"
     GET .../ab_test_sends?sent_at=gte.{{ weekStart }}&select=test_id,variant_id,converted
     fullResponse: true

  -> HTTP Request "Fetch Winners Declared This Week"
     GET .../ab_tests?winner_declared_at=gte.{{ weekStart }}&status=eq.winner_declared&select=*
     fullResponse: true

  -> Code Node "Build A/B Report"
     // For each active test:
     // - Total sends per variant this week
     // - Conversion rate per variant
     // - Progress toward min_sends_per_variant threshold
     // - Winner declared this week? Show results
     //
     // Sections:
     // - metrics: [Active Tests, Sends This Week, Winners Declared, Avg Conversion Rate]
     // - table: Test Results (test_name, channel, variant_a_rate, variant_b_rate, status, sends_remaining)
     // - highlights: New winners, tests nearing completion
     // - actions: Next tests to seed, underperforming variants

  -> Execute Sub-workflow TG-118 (build HTML)
  -> Execute Sub-workflow TG-95 (send email)
     subject: 'TG Weekly A/B Testing Report — {{ weekStart }} to {{ weekEnd }}'

  -> HTTP Request "Store Report"
     POST .../intelligence_reports
     Body: { report_type: 'ab_test_results', report_period_start: weekStart, ... }
```
