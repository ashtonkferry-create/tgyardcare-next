Create TG-85 as a standalone workflow. Quo fires call.completed webhook when a call ends — TG-85 checks if answeredAt is null (= missed call), logs it, upserts the lead, checks SMS consent, sends the auto-reply via Quo API FROM the business number.

Architecture:
- Trigger: n8n webhook (Quo POSTs here on call.completed)
- Parse: check answeredAt === null, extract from.phoneNumber as caller number
- SMS send: POST to api.openphone.com/v1/messages (FROM Quo number, replies go to Quo inbox)

Quo webhook payload structure (call.completed):
```json
{
  "type": "call.completed",
  "data": {
    "object": {
      "id": "ACxxx",
      "answeredAt": null,
      "direction": "incoming",

      "to": { "phoneNumber": "+16085356057" },
      "phoneNumberId": "PNxxx"
    }
  }
}
```
