# Plan 01-09: Brevo Email Marketing System

## Objective

Build TotalGuard's complete Brevo email marketing infrastructure: conversion-optimized HTML templates, campaign strategy for all 11 nurture sequences, and migrate all SMS nodes from Twilio credential nodes to Brevo transactional SMS HTTP calls.

## Background

The existing n8n workflows (TG-08 through TG-17) had the automation logic for email sequences but sent only plain text. The `automation/email-templates/` directory was empty. 27 workflows used Twilio credential nodes that required a paid Twilio account for SMS. This plan upgrades the full email marketing stack.

## What Was Built

### 1. HTML Email Templates (6 templates)
All templates are mobile-responsive, table-based HTML (Outlook-safe), with:
- TotalGuard brand colors (green #2D5A27, gold #C5A55A)
- Header bar with phone number, trust bar, branded footer
- Review blocks with real Google reviews (Susan McBryde, Nick Bauman, etc.)
- CTA buttons, check-item lists, dividers, signoff components

| Template | Use Case |
|----------|----------|
| `tg-welcome.html` | Welcome series email 1 — post-job onboarding |
| `tg-educational.html` | Email 2 — pro lawn care tips (educational nurture) |
| `tg-review.html` | Email 3 — Google review request |
| `tg-referral.html` | Email 4 — $25 referral incentive |
| `tg-hero.html` | Email 5 — seasonal upsell / hero offer |
| `tg-urgency.html` | Flash sales, seasonal campaigns, re-engagement |

### 2. Campaign Strategy JSON Files (10 campaigns)
Each JSON defines: subject lines A/B, preview text, headline, body HTML, review block, CTA, personalization tokens, send timing, and goal.

| Campaign File | Workflow | Emails |
|---------------|----------|--------|
| `welcome-series.json` | TG-08 | 5 emails (days 0, 7, 14, 21, 30) |
| `lead-followup.json` | TG-09 | Hot/warm/cold tracks |
| `cross-sell.json` | TG-10 | 3 emails |
| `reengagement.json` | TG-11 | 4-email win-back ladder |
| `vip-upgrade.json` | TG-12 | 2 emails |
| `review-request.json` | TG-13 | 2 emails |
| `seasonal-campaigns.json` | TG-14/15/16 | Spring (3), Fall (4), Snow (3) |
| `newsletter-yard-report.json` | TG-81 | Monthly "The Yard Report" |
| `flash-sales.json` | TG-82 | Flash sale announcements |
| `weather-triggered.json` | TG-56 | Weather-event triggered |

### 3. Twilio → Brevo SMS Migration
27 workflows migrated from `n8n-nodes-base.twilio` (requires Twilio credentials) to HTTP Request nodes targeting `https://api.brevo.com/v3/transactionalSMS/sms` with `TG_BREVO_API_KEY`. Consolidated SMS and email under a single Brevo account.

### 4. TG-08 HTML Inline Builder
TG-08's Code node now builds full HTML emails inline using a JS builder with `buildHtml()`, `checkItem()`, `reviewBlock()`, `ctaButton()`, `paragraph()`, `greeting()`, `signoff()` helpers. All 5 welcome emails generate mobile-responsive HTML.

### 5. Brevo Setup Guide
`automation/docs/brevo-setup-guide.md` covers:
- Domain authentication (DKIM, SPF, DMARC for tgyardcare.com)
- Sender configuration (2 senders)
- Contact list structure (5 lists)
- Custom attributes (8 fields)
- Frequency capping rules (2 emails per 7 days)
- Going-live checklist
