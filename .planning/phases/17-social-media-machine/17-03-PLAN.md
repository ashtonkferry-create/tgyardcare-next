  <name>Task 1: Build TG-135 daily social publishing orchestrator</name>
  <files>automation/n8n-workflows/TG-135-daily-social-publisher.json</files>
  <action>
Create a proper n8n workflow JSON for TG-135 with these nodes:

1. **Schedule Trigger** -- Fires daily at 9:55 AM CT (5 min before post time to allow processing). Cron: `55 9 * * *`.

2. **Fetch Today's Scheduled Post** (HTTP Request) -- GET from Supabase:
   URL: `https://lwtmvzhwekgdxkaisfra.supabase.co/rest/v1/social_posts?status=eq.scheduled&scheduled_for=lte.{{now_plus_1h}}&order=scheduled_for.asc&limit=1`
   Headers: apikey + Authorization with anon key.
   This fetches the next post due for publishing (scheduled_for <= now + 1 hour).

3. **IF Post Found** -- Check if result array has items. If empty, send brief Telegram: "No social post scheduled for today" and stop.

4. **Mark as Publishing** (HTTP Request) -- PATCH the post status to "publishing" to prevent double-publish:
   URL: `https://lwtmvzhwekgdxkaisfra.supabase.co/rest/v1/social_posts?id=eq.{{post_id}}`
   Body: `{"status": "publishing"}`

5. **Call TG-AUTOPOST** (Execute Workflow node) -- Call the TG-AUTOPOST-social-publisher sub-workflow with:
   - post_text: from the social_posts record
   - image_url: from the social_posts record (may be null)
   - platforms: from the social_posts record
   - post_type: from the social_posts record

6. **Check Result** (IF node) -- Check if TG-AUTOPOST returned success.

7. **On Success: Update Post** (HTTP Request) -- PATCH the post:
   URL: same as step 4
   Body: `{"status": "published", "published_at": "{{current_timestamp}}"}`

8. **On Failure: Retry Once** (Execute Workflow node) -- Call TG-AUTOPOST again. If still fails, PATCH status to "failed".

9. **Update Failed Post** (HTTP Request) -- PATCH status to "failed" if retry also failed.

10. **Send Telegram Summary** (HTTP Request) -- Notify Vance:
    - Success: "Published to {platforms}: {post_text_first_50_chars}..."
    - Failure: "FAILED to publish social post (ID: {id}). Check TG-AUTOPOST workflow."
    Use placeholders: `OWNER_TELEGRAM_CHAT_ID`, `TELEGRAM_BOT_TOKEN`.

- The Execute Workflow node should reference TG-AUTOPOST by name (Vance will set the workflow ID after import)
- Reasonable x/y node positioning
- NO real API keys -- only publishable Supabase anon key
