  <name>Task 1: Add per-query error handling to SeasonalThemeContext</name>
  <files>src/contexts/SeasonalThemeContext.tsx</files>
  <action>
Refactor the `fetchData` function in SeasonalThemeContext.tsx. Currently it uses `Promise.all` for 4 queries, which means any single failure can disrupt all results. Change to `Promise.allSettled` so each query is independent.

For each query result:
1. If `status === 'fulfilled'` and `value.data` exists — use the data
2. If `status === 'rejected'` OR `value.error` — silently keep fallback data (do NOT console.error or console.warn)

Specifically:
- `season_settings` query: On failure, keep FALLBACK_SETTINGS (already the initial state)
- `season_override` query: On failure, keep default `{ active_override: 'auto', preview_mode: false, preview_season: null }`
- `seasonal_slides` query: On failure, keep empty array (fallback slides are already used when allSlides is empty)
- `seasonal_priority_services` query: On failure, keep empty array (fallback services are already used when allServices is empty)

Remove the outer try/catch that currently does `console.warn('Failed to fetch seasonal data:', err)`. Replace with individual silent handling per query.

Also remove the `setError` state updates — errors should be invisible to users since fallback data always works.

Keep the `isLoading` state transitions (set true at start, false at end).

Do NOT change the Supabase import, the query structure, or any other part of the component. Only change how errors are handled.

<task type="auto">
  <name>Task 2: Suppress Supabase .single() 406 error on season_override</name>
  <files>src/contexts/SeasonalThemeContext.tsx</files>
  <action>
The `season_override` query uses `.limit(1).single()` which returns a 406 error if the table is empty (PGRST116: "The result contains 0 rows"). This is a known Supabase behavior — `.single()` throws when no rows exist.

Change the query from:
```
supabase.from('season_override').select('*').limit(1).single()
```
to:
```
supabase.from('season_override').select('*').limit(1).maybeSingle()
```

`.maybeSingle()` returns `null` for data instead of throwing when no rows exist. The existing null check (`if (overrideRes.data)`) already handles this correctly.

This eliminates the 406 error that fires on every page load when the season_override table has no rows.
  </action>

  </verify>
