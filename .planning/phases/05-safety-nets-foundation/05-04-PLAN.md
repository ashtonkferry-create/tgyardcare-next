  <action>
Add auth checks to the existing middleware AFTER all existing logic (410 Gone, www redirect, dynamic redirects). The auth check should ONLY run for /portal/* and /admin/* paths (excluding /admin/login).

Add this BEFORE the final `return NextResponse.next()`:

```typescript
// Auth protection for /portal/* and /admin/* (except /admin/login)
if (
  (pathname.startsWith('/portal') || pathname.startsWith('/admin')) &&
  pathname !== '/admin/login'
) {
  // Check for Supabase auth cookie presence
  const hasAuthCookie = request.cookies.getAll().some(c => c.name.includes('-auth-token'))
  if (!hasAuthCookie) {
    const loginUrl = new URL('/admin/login', request.url)
    loginUrl.searchParams.set('redirect', pathname)
    return NextResponse.redirect(loginUrl)
  }
}
```

This is simpler and doesn't require importing @supabase/ssr into middleware. The server-side auth verification happens in the actual page components — middleware just provides a quick redirect for unauthenticated users.

Place this block AFTER the dynamic redirect check and BEFORE the final `return NextResponse.next()`.

Also update the `config.matcher` to ensure /admin and /portal paths are matched. The current matcher already includes `'/((?!_next|api|admin).*)'` which EXCLUDES admin. Either:
- Change to `'/((?!_next|api).*)'` to include admin paths, OR
- Add explicit entries: `'/admin/:path*'`, `'/portal/:path*'`

Choose the explicit entries approach (safer, doesn't break existing behavior):
Add `'/admin/:path*'` and `'/portal/:path*'` to the matcher array.
