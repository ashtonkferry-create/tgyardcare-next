3. Create `.tsc-error-budget.json` in the project root with this structure:
```json

    "topErrors": [

  },
  "budget": {
    "rule": "New code in Phase 6+ must not increase errorCount above baseline",
    "checkCommand": "npx tsc --noEmit 2>&1 | grep -c 'error TS'"
  }
}
```

Fill in the actual error count and top error types from the tsc output.

If tsc produces zero errors (unlikely but possible with strict:false), record that as the baseline.
