Create n8n workflow JSON for TG-134. Listens for Telegram replies (approve/reject/edit) and processes GBP post approvals.

Workflow flow:

1. Webhook Trigger (id: tg134-webhook, name: "Telegram Approval Webhook")
   - type: n8n-nodes-base.webhook, typeVersion: 2
   - path: tg134-gbp-approval
   - httpMethod: POST
   - Note: This will receive forwarded Telegram messages from a simple Telegram bot update handler OR can be triggered manually via n8n UI

   ALTERNATIVE: Use Telegram Trigger node
   - type: n8n-nodes-base.telegramTrigger, typeVersion: 1.1
   - credentials: Telegram credential V404qLIDXjmyzNeS
   - updates: ["message"]

2. Parse Command (id: tg134-parse, name: "Parse Approval Command")
   - type: n8n-nodes-base.code, typeVersion: 2
   - Extract command from message text: approve/reject/edit + post_id + optional new text
   - Regex: /^(approve|reject|edit)\s+([a-f0-9-]+)\s*(.*)?$/i
   - If no match, ignore (return empty)

3. Route Command (id: tg134-route, name: "Route by Command Type")
   - type: n8n-nodes-base.switch, typeVersion: 3
   - Output 0: command === "approve"
   - Output 1: command === "reject"
   - Output 2: command === "edit"

4a. Approve Branch - Update Status (id: tg134-approve, name: "Mark Post Approved")
   - type: n8n-nodes-base.httpRequest, typeVersion: 4.2
   - PATCH https://lwtmvzhwekgdxkaisfra.supabase.co/rest/v1/gbp_posts?id=eq.{post_id}
   - Body: status: "approved", approved_at: now
   - Headers: apikey sb_publishable_DeX21ldoDKl3NyJeFZzR1w_eWBUkx-v

4b. Approve Branch - Check GBP API (id: tg134-check-api, name: "Check GBP API Availability")
   - type: n8n-nodes-base.code, typeVersion: 2
   - Check environment variable or config flag GBP_API_ENABLED
   - For now: ALWAYS return false (API pending verification)
   - When API arrives: flip to true and add GBP API call node

4c. Approve Branch - Format Manual Post (id: tg134-manual, name: "Format Copy-Paste Ready Post")
   - type: n8n-nodes-base.code, typeVersion: 2
   - When GBP API unavailable, format the approved post as:
     "APPROVED - READY TO POST ON GBP

     Copy this text:
     ─────────────────
     {post_content}
     ─────────────────

     Image suggestion: {image_prompt}

     Post at: business.google.com → Posts → Add update"

4d. Approve Branch - Send Confirmation (id: tg134-confirm, name: "Send Post Confirmation")
   - type: n8n-nodes-base.telegram, typeVersion: 1.2
   - credentials: V404qLIDXjmyzNeS
   - Send the formatted manual-post message OR "Published to GBP!" if API was used

5a. Reject Branch - Update Status (id: tg134-reject, name: "Mark Post Rejected")
   - PATCH gbp_posts: status: "rejected"

5b. Reject Branch - Confirm (id: tg134-reject-confirm)
   - Telegram: "Post rejected and discarded."

6a. Edit Branch - Update Content (id: tg134-edit, name: "Update Post Content")
   - PATCH gbp_posts: post_content: {new_text}, status: "pending_approval"

6b. Edit Branch - Confirm (id: tg134-edit-confirm)
   - Telegram: "Post updated. Reply 'approve {post_id}' to publish the edited version."
