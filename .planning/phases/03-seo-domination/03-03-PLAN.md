  <name>Task 1: Build and deploy TG-97 Rank Drop Detector</name>
  <files>automation/n8n-workflows/TG-97-rank-drop-detector.json</files>
  <action>
Build TG-97 as a sub-workflow triggered by TG-96 after daily GSC sync. Detects rank drops and CTR anomalies.

Workflow structure:
```
Execute Workflow Trigger (receives: { syncDate, pagesProcessed, queriesProcessed })

  -> Code Node "Calculate Date Range"
     // Current period: syncDate (2 days ago)
     // Previous period: syncDate - 7 days
     const syncDate = $json.syncDate;
     const prevDate = new Date(new Date(syncDate).getTime() - 7*24*60*60*1000).toISOString().split('T')[0];
     return [{ json: { currentDate: syncDate, previousDate: prevDate } }];

  -> HTTP Request "Fetch Current Queries"
     GET https://lwtmvzhwekgdxkaisfra.supabase.co/rest/v1/gsc_search_queries
       ?date=eq.{{ currentDate }}
       &impressions=gt.0
       &select=query,page,clicks,impressions,ctr,position
     Headers: apikey, Authorization: Bearer SERVICE_KEY
     fullResponse: true

  -> HTTP Request "Fetch Previous Queries"
     GET https://lwtmvzhwekgdxkaisfra.supabase.co/rest/v1/gsc_search_queries
       ?date=eq.{{ previousDate }}
       &impressions=gt.0
       &select=query,page,clicks,impressions,ctr,position
     Headers: apikey, Authorization: Bearer SERVICE_KEY
     fullResponse: true

  -> Code Node "Detect Rank Drops + CTR Anomalies"
     const current = $('Fetch Current Queries').first().json.body || [];
     const previous = $('Fetch Previous Queries').first().json.body || [];
     const prevMap = new Map(previous.map(p => [p.query, p]));

     const urgentAlerts = [];  // SMS-worthy
     const warnings = [];      // Weekly digest only
     const ctrAnomalies = [];  // CTR drops

     for (const cur of current) {
       const prev = prevMap.get(cur.query);
       if (!prev) continue;

       const positionDrop = cur.position - prev.position; // positive = worse ranking

       // Urgent: drop >= 10 positions AND >= 50 weekly impressions
       if (positionDrop >= 10 && prev.impressions >= 50) {
         urgentAlerts.push({
           type: 'urgent_rank_drop',
           query: cur.query,
           oldPosition: Math.round(prev.position),
           newPosition: Math.round(cur.position),
           drop: Math.round(positionDrop),
           impressions: cur.impressions
         });
       }
       // Warning: drop 5-9 positions AND >= 20 impressions
       else if (positionDrop >= 5 && prev.impressions >= 20) {
         warnings.push({
           type: 'rank_drop_warning',
           query: cur.query,
           oldPosition: Math.round(prev.position),
           newPosition: Math.round(cur.position),
           drop: Math.round(positionDrop)
         });
       }

       // CTR anomaly: >30% drop with stable impressions (both > 50)
       if (prev.impressions >= 50 && cur.impressions >= 50 && prev.ctr > 0) {
         const ctrDrop = (prev.ctr - cur.ctr) / prev.ctr;
         if (ctrDrop > 0.30) {
           ctrAnomalies.push({
             type: 'ctr_anomaly',
             query: cur.query,
             oldCtr: (prev.ctr * 100).toFixed(1) + '%',
             newCtr: (cur.ctr * 100).toFixed(1) + '%',
             dropPct: (ctrDrop * 100).toFixed(0) + '%',
             impressions: cur.impressions
           });
         }
       }
     }

     return [{ json: {
       urgentAlerts, warnings, ctrAnomalies,
       hasUrgent: urgentAlerts.length > 0,
       hasCtrAnomaly: ctrAnomalies.length > 0,
       summary: {
         queriesCompared: current.length,
         urgentCount: urgentAlerts.length,
         warningCount: warnings.length,
         ctrAnomalyCount: ctrAnomalies.length
       }
     }}];

  -> HTTP Request "Store Alert Data"
     // Store in seo_weekly_reports JSONB for weekly summary aggregation
     POST https://lwtmvzhwekgdxkaisfra.supabase.co/rest/v1/seo_weekly_reports
     Headers: apikey, Authorization: Bearer SERVICE_KEY, Content-Type: application/json
     Prefer: resolution=merge-duplicates,return=minimal
     Body: {
       "week_start": "{{ current Monday date }}",
       "report_data": { "rank_drops": urgentAlerts, "warnings": warnings, "ctr_anomalies": ctrAnomalies },
       "top_losing_keywords": urgentAlerts.concat(warnings).map(a => a.query)
     }

  -> IF "Has Urgent Alerts" (hasUrgent OR hasCtrAnomaly)

    -> Code Node "Build SMS Alert"
       const data = $json;
       const lines = [];
       if (data.urgentAlerts.length > 0) {
         lines.push('RANK DROPS:');
         data.urgentAlerts.slice(0, 3).forEach(a => {
           lines.push(`"${a.query}" dropped ${a.drop} pos (${a.oldPosition}->${a.newPosition})`);
         });
         if (data.urgentAlerts.length > 3) lines.push(`+${data.urgentAlerts.length - 3} more`);
       }
       if (data.ctrAnomalies.length > 0) {
         lines.push('CTR DROPS:');
         data.ctrAnomalies.slice(0, 2).forEach(a => {
           lines.push(`"${a.query}" CTR ${a.oldCtr}->${a.newCtr}`);
         });
       }
       const message = 'TG SEO Alert\n' + lines.join('\n');
       return [{ json: { message, phone: 'OWNER_PHONE_PLACEHOLDER' } }];

    -> Execute Sub-workflow "Send SMS via TG-94"
       workflowId: AprqI2DgQA8lehij (TG-94 unified SMS sender)
       Input: {
         to_phone: "{{ phone }}",
         message_body: "{{ message }}",
         workflow_name: "TG-97",
         message_type: "seo_alert"
       }
```

IMPORTANT NOTES:
1. TG-97 is a SUB-WORKFLOW triggered by TG-96 (not cron-scheduled). Uses executeWorkflowTrigger.
2. SMS thresholds from CONTEXT: rank drop >= 10 AND impressions >= 50 (urgent), 5-9 drop (warning only, no SMS).
3. CTR anomaly: >30% CTR drop with both periods having >= 50 impressions.
4. Warnings go to weekly digest only (stored in seo_weekly_reports for TG-102 to aggregate).
5. SMS alert is concise -- max 3 rank drops + 2 CTR anomalies to keep message short.
6. TG-94 workflow ID: AprqI2DgQA8lehij (from Phase 2).
7. Owner phone uses OWNER_PHONE_PLACEHOLDER -- real value injected at deploy time.
8. fullResponse:true on all Supabase GET requests.
9. Store alert data in seo_weekly_reports for TG-102 weekly summary to reference.

After deploying TG-97, go back to TG-96 and UPDATE the Execute Sub-workflow node with TG-97's actual workflow ID.

<task type="auto">
  <name>Task 2: Build and deploy TG-100 Index Coverage Monitor</name>
  <files>automation/n8n-workflows/TG-100-index-coverage-monitor.json</files>
  <action>
Build TG-100 as a weekly workflow that checks all key pages via the URL Inspection API and alerts on deindexing events.

Workflow structure:
```
Schedule Trigger (cron: 0 11 * * 2 = Tuesday 6 AM CT)

  -> HTTP Request "Get Known Pages"
     GET https://lwtmvzhwekgdxkaisfra.supabase.co/rest/v1/gsc_pages
       ?select=page
       &order=clicks.desc
     Headers: apikey, Authorization: Bearer SERVICE_KEY
     fullResponse: true

  -> Code Node "Deduplicate Pages"
     // gsc_pages has multiple date entries per page -- get unique pages
     const rows = $json.body || [];
     const uniquePages = [...new Set(rows.map(r => r.page))];
     return uniquePages.map(p => ({ json: { pageUrl: p } }));

  -> HTTP Request "Get Previous Coverage"
     // Get most recent check for each page to compare verdicts
     GET https://lwtmvzhwekgdxkaisfra.supabase.co/rest/v1/index_coverage_log
       ?select=page_url,verdict,check_date
       &order=check_date.desc
     Headers: apikey, Authorization: Bearer SERVICE_KEY
     fullResponse: true

  -> Code Node "Build Previous Verdict Map"
     const logs = $('Get Previous Coverage').first().json.body || [];
     const prevMap = {};
     for (const log of logs) {
       if (!prevMap[log.page_url]) prevMap[log.page_url] = log.verdict;
     }
     const pages = $('Deduplicate Pages').all().map(i => i.json);
     return pages.map(p => ({
       json: { ...p, previousVerdict: prevMap[p.pageUrl] || null }
     }));

  -> SplitInBatches (batch size: 1)

    -> HTTP Request "Inspect URL"
       POST https://searchconsole.googleapis.com/v1/urlInspection/index:inspect
       Auth: Bearer token (same as GSC -- $vars.TG_GOOGLE_SEARCH_CONSOLE_TOKEN or Google OAuth2)
       Body: {
         "inspectionUrl": "{{ $json.pageUrl }}",
         "siteUrl": "https://tgyardcare.com/",
         "languageCode": "en-US"
       }

    -> Code Node "Extract Inspection Result"
       const result = $json.inspectionResult || {};
       const indexStatus = result.indexStatusResult || {};
       const today = new Date().toISOString().split('T')[0];

       return [{ json: {
         page_url: $('SplitInBatches').first().json.pageUrl,
         check_date: today,
         verdict: indexStatus.verdict || 'VERDICT_UNSPECIFIED',
         coverage_state: indexStatus.coverageState || null,
         indexing_state: indexStatus.indexingState || null,
         last_crawl_time: indexStatus.lastCrawlTime || null,
         page_fetch_state: indexStatus.pageFetchState || null,
         robots_txt_state: indexStatus.robotsTxtState || null,
         google_canonical: indexStatus.googleCanonical || null,
         previous_verdict: $('SplitInBatches').first().json.previousVerdict
       }}];

    -> HTTP Request "Upsert Coverage Log"
       POST https://lwtmvzhwekgdxkaisfra.supabase.co/rest/v1/index_coverage_log
       Headers: apikey, Authorization: Bearer SERVICE_KEY, Content-Type: application/json
       Prefer: resolution=merge-duplicates,return=minimal
       Body: {{ $json }}

    -> Wait Node (200ms delay between API calls -- respect 600 QPM rate limit)

  -> End SplitInBatches

  -> Code Node "Detect Deindex Events"
     const results = $('Extract Inspection Result').all().map(i => i.json);
     const deindexed = results.filter(r =>
       r.previous_verdict === 'PASS' &&
       (r.verdict === 'FAIL' || r.verdict === 'NEUTRAL')
     );
     const newIssues = results.filter(r =>
       r.verdict === 'FAIL' || r.verdict === 'NEUTRAL'
     );
     return [{ json: {
       totalChecked: results.length,
       deindexed,
       hasDeindexEvents: deindexed.length > 0,
       totalIssues: newIssues.length,
       allResults: results
     }}];

  -> IF "Has Deindex Events" (hasDeindexEvents)

    -> Code Node "Build Deindex SMS"
       const data = $json;
       const lines = ['TG DEINDEX ALERT'];
       data.deindexed.forEach(d => {
         const shortUrl = d.page_url.replace('https://tgyardcare.com', '');
         lines.push(`${shortUrl}: was indexed, now ${d.verdict}`);
       });
       lines.push(`${data.totalChecked} pages checked, ${data.totalIssues} total issues`);
       return [{ json: {
         message: lines.join('\n'),
         phone: 'OWNER_PHONE_PLACEHOLDER'
       }}];

    -> Execute Sub-workflow "Send SMS via TG-94"
       workflowId: AprqI2DgQA8lehij (TG-94)
       Input: {
         to_phone: "{{ phone }}",
         message_body: "{{ message }}",
         workflow_name: "TG-100",
         message_type: "deindex_alert"
       }
```

IMPORTANT NOTES:
1. URL Inspection API: 2,000 QPD limit. TG has ~65 pages -- well within limits.
2. Add 200ms Wait between inspections to stay under 600 QPM.
3. Deindex detection: compare current verdict against previous_verdict. PASS -> FAIL or PASS -> NEUTRAL is a deindex event.
4. Only deindex events trigger SMS. New pages showing NEUTRAL (never indexed) are not alerts.
5. Use the same GSC auth token for URL Inspection API (same scope).
6. The siteUrl MUST be "https://tgyardcare.com/" (with trailing slash) for the API to accept it.
7. fullResponse:true on all Supabase GET requests.
8. PLACEHOLDER strings for all secrets.
9. TG-94 workflow ID: AprqI2DgQA8lehij.

Deploy via n8n API. Save JSON locally. Set to ACTIVE.
