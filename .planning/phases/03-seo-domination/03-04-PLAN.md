  <name>Task 1: Build and deploy TG-98 Content Gap Detector</name>
  <files>automation/n8n-workflows/TG-98-content-gap-detector.json</files>
  <action>
Build TG-98 to detect content gaps: GSC queries getting impressions but no dedicated page on tgyardcare.com. Runs weekly Monday 7 AM CT.

Workflow structure:
```
Schedule Trigger (cron: 0 12 * * 1 = Monday 7 AM CT)

  -> Code Node "Calculate Date Range"
     // Last 7 days of GSC data (with 2-day lag)
     const endDate = new Date(Date.now() - 2*24*60*60*1000).toISOString().split('T')[0];
     const startDate = new Date(Date.now() - 9*24*60*60*1000).toISOString().split('T')[0];
     return [{ json: { startDate, endDate } }];

  -> HTTP Request "Fetch Recent Query Data"
     // Get aggregate query stats for the last week
     // Use GSC API directly for weekly aggregate (more complete than daily snapshots)
     POST https://www.googleapis.com/webmasters/v3/sites/https%3A%2F%2Ftgyardcare.com/searchAnalytics/query
     Auth: Bearer token
     Body: {
       "startDate": "{{ startDate }}",
       "endDate": "{{ endDate }}",
       "dimensions": ["query"],
       "rowLimit": 25000,
       "startRow": 0,
       "dataState": "final"
     }

  -> HTTP Request "Get Existing Pages"
     // Get all pages we have content for
     GET https://lwtmvzhwekgdxkaisfra.supabase.co/rest/v1/gsc_pages?select=page&order=clicks.desc
     Headers: apikey, Authorization: Bearer SERVICE_KEY
     fullResponse: true

  -> HTTP Request "Get Existing Blog Posts"
     GET https://lwtmvzhwekgdxkaisfra.supabase.co/rest/v1/blog_posts?select=target_keyword,slug&status=eq.published
     Headers: apikey, Authorization: Bearer SERVICE_KEY
     fullResponse: true

  -> Code Node "Detect Content Gaps"
     const queries = ($('Fetch Recent Query Data').first().json.rows || []);
     const pages = ($('Get Existing Pages').first().json.body || []).map(p => p.page);
     const blogKeywords = ($('Get Existing Blog Posts').first().json.body || [])
       .map(b => b.target_keyword?.toLowerCase()).filter(Boolean);

     // Build set of topics we already cover (from page URLs and blog keywords)
     const coveredTopics = new Set([
       ...pages.map(p => p.toLowerCase()),
       ...blogKeywords
     ]);

     const today = new Date().toISOString().split('T')[0];
     const gaps = queries
       .filter(q => {
         const query = q.keys[0].toLowerCase();
         // Has meaningful impressions
         if (q.impressions < 10) return false;
         // Check if any existing page/keyword covers this query
         const hasDedicatedPage = [...coveredTopics].some(topic =>
           topic.includes(query) || query.includes(topic.replace(/https?:\/\/[^/]+\//, '').replace(/-/g, ' '))
         );
         return !hasDedicatedPage;
       })
       .map(q => ({
         query: q.keys[0],
         impressions: q.impressions,
         clicks: q.clicks,
         avg_position: Math.round(q.position * 100) / 100,
         priority_score: Math.round(q.impressions * (1 - q.position / 100) * 100) / 100,
         has_dedicated_page: false,
         suggested_page_type: q.position < 20 ? 'blog_post' : 'location_page',
         status: 'new',
         detected_at: today
       }))
       .sort((a, b) => b.priority_score - a.priority_score)
       .slice(0, 30);  // Top 30 gaps

     return [{ json: { gaps, totalGaps: gaps.length } }];

  -> IF "Has Gaps" (totalGaps > 0)

    -> HTTP Request "Upsert Content Gaps"
       POST https://lwtmvzhwekgdxkaisfra.supabase.co/rest/v1/seo_content_gaps
       Headers: apikey, Authorization: Bearer SERVICE_KEY, Content-Type: application/json
       Prefer: resolution=merge-duplicates,return=minimal
       Body: {{ gaps array }}

    -> Code Node "Build Gap Report Email"
       const gaps = $json.gaps.slice(0, 15); // Top 15 for email
       const rows = gaps.map(g =>
         `<tr><td>${g.query}</td><td>${g.impressions}</td><td>${g.avg_position}</td><td>${g.priority_score}</td><td>${g.suggested_page_type}</td></tr>`
       ).join('');

       const html = `
       <h2>Weekly Content Gap Report</h2>
       <p>${$json.totalGaps} content gaps detected from GSC data.</p>
       <table border="1" cellpadding="8">
         <tr><th>Query</th><th>Impressions</th><th>Avg Position</th><th>Priority Score</th><th>Suggested Type</th></tr>
         ${rows}
       </table>
       <p>Priority Score = impressions * (1 - position/100). Higher = more valuable.</p>
       <p>Full data in Supabase seo_content_gaps table.</p>`;

       return [{ json: {
         to_email: 'totalguardllc@gmail.com',
         subject: `SEO Content Gaps: ${$json.totalGaps} opportunities found`,
         html_body: html,
         workflow_name: 'TG-98',
         email_type: 'seo_gap_report'
       }}];

    -> Execute Sub-workflow "Send via TG-95"
       workflowId: IUDLrQrAkcLFLsIC (TG-95)
       Input: { to_email, subject, html_body, workflow_name, email_type }
```

IMPORTANT NOTES:
1. Gap detection is heuristic: match queries against page URLs and blog target keywords. Not perfect but catches obvious gaps.
2. Priority score formula: impressions * (1 - position/100). High impressions + poor ranking = big opportunity.
3. Top 30 gaps stored in seo_content_gaps table. Top 15 included in email for readability.
4. Use merge-duplicates upsert since seo_content_gaps has UNIQUE(query, detected_at).
5. fullResponse:true on all Supabase GET requests.
6. Uses GSC API directly for weekly aggregate (more complete than summing daily snapshots).
7. TG-95 workflow ID: IUDLrQrAkcLFLsIC.

Deploy via n8n API. Save JSON locally. Set to ACTIVE.

  <name>Task 2: Build and deploy TG-99 Ranking Opportunity Detector</name>
  <files>automation/n8n-workflows/TG-99-ranking-opportunity-detector.json</files>
  <action>
Build TG-99 to find quick-win ranking opportunities: queries where TG ranks 4-20 (close to page 1 but not top 3). Runs weekly Monday 7 AM CT alongside TG-98.

Workflow structure:
```
Schedule Trigger (cron: 0 12 * * 1 = Monday 7 AM CT)

  -> Code Node "Calculate Date Range"
     // Same as TG-98: last 7 days with 2-day lag
     const endDate = new Date(Date.now() - 2*24*60*60*1000).toISOString().split('T')[0];
     const startDate = new Date(Date.now() - 9*24*60*60*1000).toISOString().split('T')[0];
     return [{ json: { startDate, endDate } }];

  -> HTTP Request "Fetch Query Data with Pages"
     // Get query+page data to know WHICH page ranks for each query
     POST https://www.googleapis.com/webmasters/v3/sites/https%3A%2F%2Ftgyardcare.com/searchAnalytics/query
     Auth: Bearer token
     Body: {
       "startDate": "{{ startDate }}",
       "endDate": "{{ endDate }}",
       "dimensions": ["query", "page"],
       "rowLimit": 25000,
       "startRow": 0,
       "dataState": "final"
     }
     NOTE: Yes, this uses multi-dimension ["query", "page"]. For opportunity detection,
     knowing WHICH page ranks for a query is essential. Accept some data aggregation --
     this is analysis, not raw storage.

  -> Code Node "Find Quick Wins"
     const rows = $json.rows || [];
     const opportunities = rows
       .filter(r => {
         const position = r.position;
         const impressions = r.impressions;
         // Quick wins: position 4-20 AND meaningful impressions
         return position >= 4 && position <= 20 && impressions >= 10;
       })
       .map(r => ({
         query: r.keys[0],
         page: r.keys[1],
         position: Math.round(r.position * 10) / 10,
         impressions: r.impressions,
         clicks: r.clicks,
         ctr: Math.round(r.ctr * 10000) / 10000,
         potential_score: Math.round(r.impressions * (1 - r.position / 100) * 100) / 100
       }))
       .sort((a, b) => b.potential_score - a.potential_score)
       .slice(0, 25);  // Top 25 opportunities

     return [{ json: { opportunities, totalFound: opportunities.length } }];

  -> IF "Has Opportunities" (totalFound > 0)

    -> Code Node "Build Opportunity Report Email"
       const opps = $json.opportunities.slice(0, 15);
       const rows = opps.map(o => {
         const shortPage = o.page.replace('https://tgyardcare.com', '');
         return `<tr><td>${o.query}</td><td>${shortPage}</td><td>${o.position}</td><td>${o.impressions}</td><td>${o.clicks}</td><td>${o.potential_score}</td></tr>`;
       }).join('');

       const html = `
       <h2>Weekly Ranking Opportunities</h2>
       <p>${$json.totalFound} quick-win opportunities (position 4-20 with meaningful impressions).</p>
       <p>These queries are close to top 3 — small improvements yield big traffic gains.</p>
       <table border="1" cellpadding="8">
         <tr><th>Query</th><th>Page</th><th>Position</th><th>Impressions</th><th>Clicks</th><th>Potential</th></tr>
         ${rows}
       </table>
       <p><strong>Action:</strong> Optimize title tags, add internal links, expand content for these queries.</p>`;

       return [{ json: {
         to_email: 'totalguardllc@gmail.com',
         subject: `SEO Quick Wins: ${$json.totalFound} ranking opportunities`,
         html_body: html,
         workflow_name: 'TG-99',
         email_type: 'seo_opportunity_report'
       }}];

    -> Execute Sub-workflow "Send via TG-95"
       workflowId: IUDLrQrAkcLFLsIC (TG-95)
       Input: { to_email, subject, html_body, workflow_name, email_type }
```

IMPORTANT NOTES:
1. This workflow intentionally uses ["query", "page"] dimensions to know which page ranks for each query. Accept the data aggregation tradeoff -- this is analysis, not storage.
2. Quick wins = position 4-20 (page 1-2 but not top 3) with impressions >= 10.
3. Potential score same formula as content gaps: impressions * (1 - position/100).
4. No storage to seo_content_gaps -- these are existing pages that need optimization, not new pages to create. Report-only.
5. TG-95 workflow ID: IUDLrQrAkcLFLsIC.
6. fullResponse:true on any Supabase GET.

Deploy via n8n API. Save JSON locally. Set to ACTIVE.

2. Queries GSC API with ["query", "page"] dimensions
3. Filters for position 4-20 with impressions >= 10
4. Sorts by potential_score descending
5. Sends email report via TG-95

  <name>Task 3: Build and deploy TG-101 Content Staleness Checker</name>
  <files>automation/n8n-workflows/TG-101-content-staleness-checker.json</files>
  <action>
Build TG-101 to identify stale content (not updated in 6+ months). Runs weekly Sunday midnight CT.

Workflow structure:
```
Schedule Trigger (cron: 0 5 * * 0 = Sunday midnight CT)

  -> Code Node "Calculate Staleness Threshold"
     const sixMonthsAgo = new Date(Date.now() - 180*24*60*60*1000).toISOString();
     return [{ json: { staleDate: sixMonthsAgo } }];

  -> HTTP Request "Get Stale Blog Posts"
     GET https://lwtmvzhwekgdxkaisfra.supabase.co/rest/v1/blog_posts
       ?updated_at=lt.{{ staleDate }}
       &status=eq.published
       &select=id,title,slug,updated_at,target_keyword,category
     Headers: apikey, Authorization: Bearer SERVICE_KEY
     fullResponse: true

  -> HTTP Request "Get Stale Location Pages"
     GET https://lwtmvzhwekgdxkaisfra.supabase.co/rest/v1/seo_location_pages
       ?updated_at=lt.{{ staleDate }}
       &status=eq.published
       &select=id,city,service_type,slug,updated_at
     Headers: apikey, Authorization: Bearer SERVICE_KEY
     fullResponse: true

  -> Code Node "Compile Staleness Report"
     const stalePosts = $('Get Stale Blog Posts').first().json.body || [];
     const staleLocations = $('Get Stale Location Pages').first().json.body || [];
     const totalStale = stalePosts.length + staleLocations.length;

     if (totalStale === 0) return [{ json: { skip: true, message: 'No stale content' } }];

     return [{ json: {
       stalePosts,
       staleLocations,
       totalStale,
       skip: false
     }}];

  -> IF "Has Stale Content" (skip !== true)

    -> Code Node "Build Staleness Report Email"
       const data = $json;
       let html = `<h2>Content Staleness Report</h2>
       <p>${data.totalStale} pieces of content haven't been updated in 6+ months.</p>`;

       if (data.stalePosts.length > 0) {
         html += '<h3>Stale Blog Posts</h3><table border="1" cellpadding="8"><tr><th>Title</th><th>Last Updated</th><th>Target Keyword</th></tr>';
         data.stalePosts.forEach(p => {
           const updated = new Date(p.updated_at).toLocaleDateString();
           html += `<tr><td>${p.title}</td><td>${updated}</td><td>${p.target_keyword || 'none'}</td></tr>`;
         });
         html += '</table>';
       }

       if (data.staleLocations.length > 0) {
         html += '<h3>Stale Location Pages</h3><table border="1" cellpadding="8"><tr><th>City</th><th>Service</th><th>Last Updated</th></tr>';
         data.staleLocations.forEach(p => {
           const updated = new Date(p.updated_at).toLocaleDateString();
           html += `<tr><td>${p.city}</td><td>${p.service_type}</td><td>${updated}</td></tr>`;
         });
         html += '</table>';
       }

       html += '<p><strong>Action:</strong> Refresh content with current information, seasonal updates, or new local details.</p>';

       return [{ json: {
         to_email: 'totalguardllc@gmail.com',
         subject: `Content Refresh Needed: ${data.totalStale} stale page(s)`,
         html_body: html,
         workflow_name: 'TG-101',
         email_type: 'content_staleness_report'
       }}];

    -> Execute Sub-workflow "Send via TG-95"
       workflowId: IUDLrQrAkcLFLsIC (TG-95)
       Input: { to_email, subject, html_body, workflow_name, email_type }
```

IMPORTANT NOTES:
1. Staleness threshold: 6 months (180 days) since last updated_at.
2. Checks both blog_posts and seo_location_pages tables.
3. Only checks published content (drafts don't need freshness monitoring).
4. Report-only workflow -- does not modify any content.
5. Skips email entirely if no stale content found (no empty reports).
6. fullResponse:true on all Supabase GET requests.
7. TG-95 workflow ID: IUDLrQrAkcLFLsIC.
8. If updated_at column doesn't exist on blog_posts, fall back to published_at or created_at.

Deploy via n8n API. Save JSON locally. Set to ACTIVE.
