     // Compare target cities against existing pages

     const needed = cities.filter(c => !existingCities.has(c.city));

       // Use the city-specific prompt template from RESEARCH.md

<task type="auto">
  <name>Task 2: Build and deploy TG-104 Content Quality Checker</name>
  <files>automation/n8n-workflows/TG-104-content-quality-checker.json</files>
  <action>
Build TG-104 as a sub-workflow triggered by TG-103 that validates recently generated content drafts for quality.

Workflow structure:
```
Execute Workflow Trigger (receives: { generationRun: true })

  -> HTTP Request "Get Recent Drafts"
     GET https://lwtmvzhwekgdxkaisfra.supabase.co/rest/v1/blog_posts
       ?status=eq.draft
       &ai_generated=eq.true
       &created_at=gte.{{ 24 hours ago ISO }}
       &select=id,title,content,slug,word_count,city,target_keyword
     Headers: apikey, Authorization: Bearer SERVICE_KEY
     fullResponse: true

  -> Code Node "Validate Quality"
     const drafts = $json.body || [];
     const issues = [];
     const passed = [];

     for (const draft of drafts) {
       const content = draft.content || '';
       const plainText = content.replace(/<[^>]*>/g, '');
       const wordCount = plainText.split(/\s+/).filter(w => w).length;
       const problems = [];

       // Check 1: Minimum word count
       if (wordCount < 800) problems.push(`Word count ${wordCount} < 800 minimum`);

       // Check 2: No placeholder text
       if (/\[.*?\]/.test(content) && /neighborhood|city|park/i.test(content.match(/\[.*?\]/)?.[0] || ''))
         problems.push('Contains placeholder text in brackets');

       // Check 3: Has H2 headings (structural quality)
       const h2Count = (content.match(/<h2/gi) || []).length;
       if (h2Count < 3) problems.push(`Only ${h2Count} H2 headings (need 3+)`);

       // Check 4: Mentions Wisconsin/Dane County/Zone 5a (local relevance)
       const localTerms = ['Wisconsin', 'Dane County', 'Zone 5a', 'silt loam', 'Madison'];
       const localHits = localTerms.filter(t => plainText.toLowerCase().includes(t.toLowerCase()));
       if (localHits.length < 2) problems.push(`Only ${localHits.length} local references (need 2+)`);

       // Check 5: Has CTA
       if (!plainText.includes('608') && !plainText.includes('tgyardcare'))
         problems.push('Missing TotalGuard CTA (no phone or website)');

       if (problems.length > 0) {
         issues.push({ id: draft.id, title: draft.title, slug: draft.slug, problems });
       } else {
         passed.push({ id: draft.id, title: draft.title });
       }
     }

     return [{ json: { issues, passed, totalChecked: drafts.length, totalPassed: passed.length, totalFailed: issues.length } }];

  -> IF "Has Quality Issues" (issues.length > 0)

    -> Code Node "Build Quality Report Email"
       const data = $json;
       const issueRows = data.issues.map(i =>
         `<tr><td>${i.title}</td><td><ul>${i.problems.map(p => '<li>' + p + '</li>').join('')}</ul></td></tr>`
       ).join('');

       const html = `
       <h2>SEO Content Quality Report</h2>
       <p>${data.totalChecked} drafts checked: ${data.totalPassed} passed, ${data.totalFailed} need review.</p>
       <table border="1" cellpadding="8">
         <tr><th>Draft</th><th>Issues</th></tr>
         ${issueRows}
       </table>
       <p>Review these drafts in Supabase blog_posts table (status='draft').</p>`;

       return [{ json: {
         to_email: 'totalguardllc@gmail.com',
         subject: `SEO Content Quality: ${data.totalFailed} draft(s) need review`,
         html_body: html,
         workflow_name: 'TG-104',
         email_type: 'seo_quality_report'
       }}];

    -> Execute Sub-workflow "Send Report via TG-95"
       workflowId: IUDLrQrAkcLFLsIC (TG-95 unified email sender)
       Input: { to_email, subject, html_body, workflow_name, email_type }
```

IMPORTANT NOTES:
1. Quality checks are heuristic -- they flag obvious issues, not replace human judgment.
2. Word count check strips HTML tags before counting.
3. Placeholder detection looks for [bracket text] that contains location-related words.
4. Local relevance check ensures content actually mentions Wisconsin/local details.
5. All flagged drafts remain in blog_posts as 'draft' -- the report just alerts the owner.
6. TG-95 workflow ID is known: IUDLrQrAkcLFLsIC (from Phase 2).
7. fullResponse:true on the Supabase GET for drafts.
8. Owner email: totalguardllc@gmail.com

Deploy via n8n API. Save JSON locally. Set to ACTIVE (it's a sub-workflow, only runs when called).
