# Plan 08-02: Customer Portal Auth Flow

## Goal
Build the magic link login flow for the customer portal. Customer enters email → receives magic link → clicks → lands on dashboard. Fully separate from admin login.

## Files to Create/Modify

### src/app/portal/login/page.tsx
Full-screen magic link login page matching TotalGuard dark brand.
- Background: #050d07, green accent #22c55e
- Logo + headline: "Your TotalGuard Portal"
- Email input + "Send Magic Link" button (with loading state)
- Uses `createClient()` from `@/lib/supabase/client` (browser client)
- Calls `supabase.auth.signInWithOtp({ email, options: { emailRedirectTo: origin + '/portal/auth/callback' } })`
- On success: shows confirmation message ("Check your email — link expires in 10 minutes")
- Motion animations: fade in, blur-fade stagger

### src/app/portal/auth/callback/route.ts
Server-side route handler for Supabase OTP exchange.
- GET handler: reads `?code=` param
- Uses `createServerClient` from `@supabase/ssr` (with cookie handling)
- Calls `supabase.auth.exchangeCodeForSession(code)`
- On success: redirects to `/portal/dashboard` (or `redirect` param if present)
- On failure: redirects to `/portal/login?error=invalid_link`
- Must set cookies properly for session persistence

### src/middleware.ts (update)
Update portal protection to redirect to `/portal/login` instead of `/admin/login`.
Change:
```typescript
if (pathname.startsWith('/portal') && pathname !== '/admin/login') {
  ...
  loginUrl = new URL('/admin/login', request.url)
```
To:
```typescript
if (pathname.startsWith('/portal') &&
    !pathname.startsWith('/portal/login') &&
    !pathname.startsWith('/portal/auth')) {
  ...
  loginUrl = new URL('/portal/login', request.url)
```
