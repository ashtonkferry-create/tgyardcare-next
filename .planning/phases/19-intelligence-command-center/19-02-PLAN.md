--- TG-139: Real-time Lead Alert ---

NOTE: TG-01 already sends a Telegram alert for new leads (Phase 12). TG-139 is a SUPPLEMENTARY enriched alert that adds lead scoring context and competitive intelligence. If TG-01's existing Telegram alert is sufficient, TG-139 can serve as a DEDUP/ENHANCEMENT layer.

APPROACH: Instead of duplicating TG-01's alert, make TG-139 a webhook that TG-01 calls AFTER its initial processing. TG-139 enriches with score tier and sends a formatted intelligence-style alert.

Workflow flow:

1. Webhook Trigger (id: tg139-webhook, name: "New Lead Intelligence Alert")
   - type: n8n-nodes-base.webhook, typeVersion: 2
   - path: tg139-lead-alert
   - httpMethod: POST
   - Expects: name, email, phone, source, service_type, message, lead_score, lead_tier

2. Enrich Lead Context (id: tg139-enrich, name: "Enrich Lead Context")
   - type: n8n-nodes-base.code, typeVersion: 2
   - Check if lead is from a served zip code
   - Determine urgency: HOT (score 80+), WARM (50-79), COOL (<50)
   - Check for repeat lead (same phone in leads table last 90 days)
   - Flag high-value services (hardscaping, gutter guard = high AOV)

3. Format Alert (id: tg139-format, name: "Format Lead Intelligence Alert")
   - type: n8n-nodes-base.code, typeVersion: 2
   - "{urgency_emoji} NEW LEAD ALERT

     {name} | {phone}
     Source: {source}
     Service: {service_type}
     Score: {lead_score} ({lead_tier})
     {if repeat: 'REPEAT LEAD - contacted {days} days ago'}
     {if high_aov: 'HIGH VALUE SERVICE'}

     Message: {message}

     Response deadline: {5_min_from_now}"

4. Send Alert (id: tg139-send, name: "Send Lead Alert")
   - type: n8n-nodes-base.telegram, typeVersion: 1.2
   - credentials: V404qLIDXjmyzNeS
   - chatId: OWNER_TELEGRAM_CHAT_ID

Connections: tg139-webhook -> tg139-enrich -> tg139-format -> tg139-send

--- TG-140: Review Alert ---

Workflow flow:

1. Schedule Trigger (id: tg140-trigger, name: "Hourly Review Check")
   - type: n8n-nodes-base.scheduleTrigger, typeVersion: 1.2
   - rule: every hour at :30 (offset from other workflows)
   - NOTE: If TG-131 (Phase 14) exists and calls TG-140, this schedule is a fallback

   ALTERNATIVE: Webhook trigger if TG-131 calls it directly
   - Also include webhook path: tg140-review-alert

2. Fetch New Reviews (id: tg140-fetch, name: "Fetch New Google Reviews")
   - type: n8n-nodes-base.httpRequest, typeVersion: 4.2
   - GET https://maps.googleapis.com/maps/api/place/details/json
   - place_id: ChIJMQEDNR3apwMRFbkOyDYmDAE
   - fields: reviews
   - key: GOOGLE_PLACES_API_KEY_PLACEHOLDER
   - continueOnFail: true

3. Check for New (id: tg140-check, name: "Compare Against Known Reviews")
   - type: n8n-nodes-base.code, typeVersion: 2
   - GET google_reviews from Supabase (last known reviews)
   - Compare author names + timestamps to find truly new reviews
   - If no new reviews, stop execution (return empty)
   - If google_reviews table missing, compare against last_review_check timestamp

4. Classify Severity (id: tg140-classify, name: "Classify Review Severity")
   - type: n8n-nodes-base.code, typeVersion: 2
   - 5 stars: INFO severity, positive emoji
   - 4 stars: INFO severity
   - 3 stars: WARNING severity
   - 1-2 stars: CRITICAL severity, needs immediate response

5. Store Review (id: tg140-store, name: "Store Review in Supabase")
   - type: n8n-nodes-base.httpRequest, typeVersion: 4.2
   - POST google_reviews: author, rating, text, review_time, severity
   - continueOnFail: true

6. Route by Severity (id: tg140-route, name: "Route by Review Severity")
   - type: n8n-nodes-base.switch, typeVersion: 3
   - Output 0: CRITICAL (1-2 stars)
   - Output 1: WARNING (3 stars)
   - Output 2: INFO (4-5 stars)

7a. Critical Path (id: tg140-critical, name: "CRITICAL Review Alert via TG-113")
   - type: n8n-nodes-base.executeWorkflow, typeVersion: 1
   - workflowId: "GHL1BUPFZL8Ic6Bc" (TG-113 critical alerts)
   - Pass: alert_type: "negative_review", message: formatted review details

7b. Normal Path (id: tg140-normal, name: "Send Review Alert to Telegram")
   - type: n8n-nodes-base.telegram, typeVersion: 1.2
   - credentials: V404qLIDXjmyzNeS
   - "{severity} NEW GOOGLE REVIEW

     {rating_stars}
     By: {author_name}

     '{review_text}'

     {if critical: 'RESPOND IMMEDIATELY at business.google.com'}
     {if 5_star: 'Consider sending referral request!'}"

Connections:
  tg140-trigger -> tg140-fetch -> tg140-check -> tg140-classify -> tg140-store -> tg140-route
  tg140-route (output 0) -> tg140-critical
  tg140-route (output 1,2) -> tg140-normal

Workflow flow:

   - rule: every 5 minutes

   - type: n8n-nodes-base.httpRequest, typeVersion: 4.2
   - GET https://tgyardcare.app.n8n.cloud/api/v1/executions?limit=50&status=error
   - Headers: X-N8N-API-KEY N8N_API_KEY_PLACEHOLDER
   - continueOnFail: true

   - type: n8n-nodes-base.code, typeVersion: 2

   - Filter executions to only those after last check
   - If no new errors, stop (return empty)
   - Update last_check timestamp

4. Classify Error Severity (id: tg141-classify, name: "Classify Error Severity")

5. Route by Severity (id: tg141-route, name: "Route by Error Severity")
   - type: n8n-nodes-base.switch, typeVersion: 3
   - Output 0: CRITICAL
   - Output 1: WARNING
   - Output 2: INFO

6a. Critical Alert (id: tg141-critical, name: "CRITICAL System Alert via TG-113")
   - type: n8n-nodes-base.executeWorkflow, typeVersion: 1
   - workflowId: "GHL1BUPFZL8Ic6Bc" (TG-113)
   - Pass: alert_type: "system_error", workflow_name, error_message, error_count

6b. Warning/Info Alert (id: tg141-alert, name: "Send System Alert to Telegram")
   - type: n8n-nodes-base.telegram, typeVersion: 1.2
   - credentials: V404qLIDXjmyzNeS
   - "{severity} SYSTEM ALERT

     Workflow: {workflow_name} (TG-{number})
     Error: {error_message}
     Count: {error_count} in last hour

     {if critical: 'LEADS MAY BE DROPPING - check immediately'}

     View: https://tgyardcare.app.n8n.cloud/workflow/{id}/executions"

7. Log Health Check (id: tg141-log, name: "Log Health Check Result")
   - type: n8n-nodes-base.httpRequest, typeVersion: 4.2
   - POST system_health_checks: checked_at, error_count, critical_count, workflows_checked

Connections:
  tg141-trigger -> tg141-fetch -> tg141-filter -> tg141-classify -> tg141-route
