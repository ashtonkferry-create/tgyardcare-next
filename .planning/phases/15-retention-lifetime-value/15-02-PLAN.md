**TG-32 Seasonal Transition Router:**

Rebuild TG-32-seasonal-transition.json as the master seasonal campaign controller. Runs on the 1st of each month, checks the current month, and triggers the appropriate seasonal campaign.

Workflow structure:

1. **Schedule Trigger** (scheduleTrigger, typeVersion 1.2)
   - Cron: `0 14 1 * *` (9am CT on the 1st of every month)
   - ID: schedule-trigger

2. **Determine Season** (code node, typeVersion 2)
   - ID: determine-season
   - Code:
     ```javascript
     const month = new Date().getMonth() + 1; // 1-12
     let season = 'none';
     let campaign = null;

     if (month === 3) { season = 'spring'; campaign = 'TG-14'; }
     else if (month === 9) { season = 'fall'; campaign = 'TG-15'; }
     else if (month === 11) { season = 'snow'; campaign = 'TG-16'; }

     return [{ json: { month, season, campaign } }];
     ```

3. **IF Campaign Due** (if node, typeVersion 2.2)
   - Condition: `$json.campaign` is not empty
   - ID: if-campaign-due

4. **Telegram Notification** (n8n-nodes-base.telegram, typeVersion 1.2)
   - chatId: use credential V404qLIDXjmyzNeS
   - text: "Seasonal campaign launching! Season: {season}, triggering {campaign}"
   - ID: telegram-notify

5. **Execute Seasonal Campaign** (executeWorkflow, typeVersion 1.1)
   - workflowId: dynamically set from $json.campaign workflow ID lookup
   - Since executeWorkflow needs a static ID, use a Switch node instead:

   Alternative: **Route to Campaign** (switch node, typeVersion 3)
   - ID: route-campaign
   - Route on `$json.campaign`:
     - "TG-14" → Execute TG-14 node
     - "TG-15" → Execute TG-15 node
     - "TG-16" → Execute TG-16 node

   Then three executeWorkflow nodes:
   - **Run TG-14** (executeWorkflow): workflowId = TG-14's n8n ID (look up during deployment)
   - **Run TG-15** (executeWorkflow): workflowId = TG-15's n8n ID
   - **Run TG-16** (executeWorkflow): workflowId = TG-16's n8n ID

**TG-14 Spring Nurture Campaign:**

Rebuild TG-14-spring-nurture.json. When triggered (by TG-32 in March, or manually), fetches all active customers and sends spring campaign email via TG-95.

Workflow structure:

1. **Execute Workflow Trigger** (executeWorkflowTrigger, typeVersion 1.1)
   - ID: trigger
   - (Allows TG-32 to call this, or can be triggered manually)

2. **Fetch Active Customers** (httpRequest, typeVersion 4.2)
   - GET `={{$vars.TG_SUPABASE_URL}}/rest/v1/leads?status=eq.customer&select=id,first_name,last_name,email,phone,service,address`
   - Also try: `status=in.(customer,active,converted)` to catch various status values
   - Headers: apikey, Authorization
   - ID: fetch-customers

3. **Filter Has Email** (code node, typeVersion 2)
   - Filter out customers without email
   - ID: filter-email

4. **Build Spring Email** (code node, typeVersion 2)
   - ID: build-email
   - For each customer, build branded HTML email:
     - Subject: "Spring is here, {name}! Time to wake up your lawn"
     - Headline: "Your Lawn is Ready for Spring"
     - Body: Spring cleanup benefits, early bird pricing mention, fertilization upsell, aeration recommendation
     - Services promoted: Spring Cleanup, Fertilization (5-step program), Aeration & Overseeding, Garden Bed Cleanup
     - Include a real 5-star review (fetch from reviews table or hardcode a great one)
     - CTA: "Schedule My Spring Cleanup" linking to tgyardcare.com/contact
     - Use the same branded HTML template pattern from TG-09 (green header, gold accents, dark footer)
   - Output: to_email, subject, html_body, workflow_name: 'TG-14', email_type: 'spring_campaign'

5. **Send Email via TG-95** (executeWorkflow, typeVersion 1.1)
   - workflowId: "FSRv5y8YzMjpyBtc" (TG-95 unified email sender)
   - workflowInputs: to_email, subject, html_body, workflow_name, email_type
   - ID: send-email

6. **Log Campaign Send** (httpRequest, typeVersion 4.2)
   - POST to Supabase campaign_sends or similar tracking table
   - continueOnFail: true (table may not exist)
   - ID: log-send

7. **Telegram Summary** (n8n-nodes-base.telegram, typeVersion 1.2)
   - After all sends, report count: "Spring campaign sent to {N} customers"
   - ID: telegram-summary

Connections:
- Trigger → Fetch Customers → Filter Email → Build Email → Send via TG-95 → Log Send
- After all items processed → Telegram Summary

**IMPORTANT:** Use TG-95 (FSRv5y8YzMjpyBtc) for email, NOT Brevo directly. TG-95 handles send logging and rate limiting.

**IMPORTANT:** For bulk campaigns (potentially 50+ customers), add a 2-second Wait node between sends to avoid rate limiting. Or use Brevo's campaign API for bulk sends — but TG-95 gives better tracking. Start with TG-95 and note if volume becomes an issue.
