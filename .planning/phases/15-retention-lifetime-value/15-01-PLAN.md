  <action>
Rebuild TG-22-review-to-referral.json to detect new 5-star Google reviews and automatically send a referral request SMS to the reviewer.

**Trigger mechanism:**
- Schedule Trigger: every 30 minutes (cron `*/30 * * * *`)
- Fetch recent 5-star reviews from Supabase `reviews` table created in last 2 hours
- Filter: `rating=eq.5&created_at=gte.{2_hours_ago}&select=id,author_name,text,rating,created_at`
- Cross-reference against `referral_requests` table (or a `referral_sms_sent` boolean field) to avoid duplicate sends

**Workflow structure:**

1. **Schedule Trigger** (scheduleTrigger, typeVersion 1.2)
   - Cron: `*/30 * * * *` (every 30 min)
   - ID: schedule-trigger

2. **Fetch Recent 5-Star Reviews** (httpRequest, typeVersion 4.2)
   - GET `={{$vars.TG_SUPABASE_URL}}/rest/v1/reviews?rating=eq.5&created_at=gte.{{ new Date(Date.now()-2*60*60*1000).toISOString() }}&select=id,author_name,text,rating,created_at,reviewer_phone,referral_sms_sent`
   - Headers: apikey, Authorization with $vars
   - ID: fetch-reviews

3. **Filter Unsent** (code node, typeVersion 2)
   - ID: filter-unsent
   - Filter out reviews where `referral_sms_sent` is true or `reviewer_phone` is missing
   - If no reviews remain, return empty (workflow ends)
   - Code: filter reviews, look up matching customer by name in `customers` or `leads` table if reviewer_phone not on review

4. **Lookup Customer Phone** (httpRequest, typeVersion 4.2)
   - For each review without reviewer_phone, search customers/leads by name
   - GET `={{$vars.TG_SUPABASE_URL}}/rest/v1/leads?or=(first_name.ilike.*{{ reviewerFirstName }}*,last_name.ilike.*{{ reviewerLastName }}*)&select=id,first_name,last_name,phone&limit=1`
   - ID: lookup-customer
   - This may not find everyone — that's OK, we only send to matches

5. **Build Referral SMS** (code node, typeVersion 2)
   - ID: build-referral-sms
   - Message: "Hi {name}! Thank you SO much for your amazing 5-star review! It means the world to us. Know someone who needs yard care? Send them our way and we'll thank you with a discount on your next service! Just have them mention your name when they call (608) 535-6057 or visit tgyardcare.com. — TotalGuard"
   - Output: to_phone, message_body, workflow_name: 'TG-22', message_type: 'referral_request', review_id

6. **Send SMS via TG-94** (executeWorkflow, typeVersion 1.1)
   - workflowId: "AprqI2DgQA8lehij"
   - workflowInputs: to_phone, message_body, workflow_name, message_type
   - ID: send-referral-sms

7. **Mark Review as Sent** (httpRequest, typeVersion 4.2)
   - PATCH `={{$vars.TG_SUPABASE_URL}}/rest/v1/reviews?id=eq.{{$json.review_id}}`
   - Body: `{ "referral_sms_sent": true }`
   - ID: mark-sent
   - continueOnFail: true (field may not exist — note in SUMMARY as manual step)

8. **Telegram Notification** (n8n-nodes-base.telegram, typeVersion 1.2)
   - chatId: use credential V404qLIDXjmyzNeS
   - text: "Referral request sent to {name} after their 5-star review!"
   - ID: telegram-notify

**Connections:**
- Schedule → Fetch Reviews → Filter Unsent → Lookup Customer → Build SMS → Send SMS → Mark Sent → Telegram Notify

**Schema note:** The `reviews` table may not have `reviewer_phone` or `referral_sms_sent` columns. The executor should:

3. Workflow degrades gracefully — uses customer/lead lookup as fallback for phone

**Workflow structure:**

1. **Webhook Trigger** (webhook, typeVersion 2)
   - Path: `tg34-referral-log`
   - Method: POST
   - ID: webhook-trigger
   - Accepts: { referrer_name, new_lead_name, new_lead_phone, service }

2. **Find Referrer** (httpRequest, typeVersion 4.2)
   - Search leads/customers by referrer_name
   - GET `={{$vars.TG_SUPABASE_URL}}/rest/v1/leads?or=(first_name.ilike.*{{ referrerFirst }}*,last_name.ilike.*{{ referrerLast }}*)&select=id,first_name,last_name,phone&limit=1`
   - ID: find-referrer

3. **Log Referral** (httpRequest, typeVersion 4.2)
   - POST `={{$vars.TG_SUPABASE_URL}}/rest/v1/referrals`
   - Body: `{ "referrer_lead_id": referrer.id, "referred_name": new_lead_name, "referred_phone": new_lead_phone, "service": service, "status": "pending", "created_at": now }`
   - ID: log-referral
   - continueOnFail: true (referrals table may not exist)

4. **Thank Referrer SMS** (executeWorkflow, typeVersion 1.1)
   - workflowId: "AprqI2DgQA8lehij" (TG-94)
   - Message: "Hi {referrer_name}! Great news — {new_lead_name} just mentioned you when they reached out about {service}! Thanks for spreading the word. Your discount will be applied to your next service. — TotalGuard"
   - ID: thank-referrer

5. **Telegram Alert** (n8n-nodes-base.telegram, typeVersion 1.2)
   - chatId: use credential V404qLIDXjmyzNeS
   - text: "New referral! {referrer_name} referred {new_lead_name} for {service}"
   - ID: telegram-alert

**Connections:**
- Webhook → Find Referrer → Log Referral → Thank Referrer SMS → Telegram Alert

**Schema note:** A `referrals` table may not exist. If missing, note SQL for Vance:
```sql

  referred_phone text,
  service text,
  status text DEFAULT 'pending',
  created_at timestamptz DEFAULT now()
);
```
