Create n8n workflow JSON for TG-131-google-review-poller. Polls Google Places API (New) every 6 hours, detects new reviews, syncs to Supabase, triggers TG-132 for each new review.

IMPORTANT: Use the Google Places API (New) -- the Places Details endpoint. This does NOT require GBP verification. The API key is stored in n8n credentials (TotalGuard Google Places). For the workflow use HTTP Request with the API key as a header.

Place ID: ChIJMQEDNR3apwMRFbkOyDYmDAE

Workflow flow:

1. Schedule Trigger (id: tg131-trigger, name: "Every 6 Hours Review Check")
   - type: n8n-nodes-base.scheduleTrigger, typeVersion: 1.2
   - rule: every 6 hours (0:00, 6:00, 12:00, 18:00 UTC)

2. Fetch Place Details (id: tg131-fetch, name: "Fetch Google Place Details with Reviews")
   - type: n8n-nodes-base.httpRequest, typeVersion: 4.2
   - GET https://places.googleapis.com/v1/places/ChIJMQEDNR3apwMRFbkOyDYmDAE
   - Headers:
     - X-Goog-Api-Key: GOOGLE_PLACES_API_KEY_PLACEHOLDER
     - X-Goog-FieldMask: reviews,rating,userRatingCount
   - The Places API (New) returns up to 5 most recent reviews

3. Extract Reviews (id: tg131-extract, name: "Extract Review Data")
   - type: n8n-nodes-base.code, typeVersion: 2
   - Parse the reviews array from the API response
   - For each review extract: author_name (from authorAttribution.displayName), rating, text (from text.text or originalText.text), relative_publish_time (from relativePublishTimeDescription), publish_time (from publishTime), review_id (hash of author+publishTime for dedup)
   - Also extract overall rating and userRatingCount for tracking
   - Return array of review objects

4. Check If New (id: tg131-dedup, name: "Filter Out Already-Synced Reviews")
   - type: n8n-nodes-base.code, typeVersion: 2
   - For each review, check google_reviews table by review_id
   - HTTP GET to Supabase: google_reviews?review_id=eq.{review_id}&select=id
   - If no rows: this is a new review, include it
   - If rows exist: already synced, skip

5. Save to Supabase (id: tg131-save, name: "Save New Review to Supabase")
   - type: n8n-nodes-base.httpRequest, typeVersion: 4.2
   - POST https://lwtmvzhwekgdxkaisfra.supabase.co/rest/v1/google_reviews
   - Body: review_id, author_name, rating, review_text, publish_time, overall_rating, total_reviews, is_five_star (rating==5), needs_response: true, social_media_queued: false
   - Prefer: return=representation
   - continueOnFail: true

6. Route by Rating (id: tg131-switch, name: "Route by Star Rating")
   - type: n8n-nodes-base.switch, typeVersion: 3
   - Rules:
     - rating <= 3 -> output 0 (negative/concerning)
     - rating == 5 -> output 1 (five star -- social proof candidate)
     - default -> output 2 (4-star, normal)

7. Send Negative Alert (id: tg131-neg-alert, name: "Send URGENT Negative Review Alert")
   - type: n8n-nodes-base.executeWorkflow, typeVersion: 1
   - workflowId: "GHL1BUPFZL8Ic6Bc" (TG-113 critical alert router)
   - Payload: alert_type: "negative_google_review", severity: "CRITICAL", title with rating stars, message with author and review text

8. Call TG-132 for Response Draft (id: tg131-call-drafter, name: "Draft AI Response via TG-132")
   - type: n8n-nodes-base.executeWorkflow, typeVersion: 1
   - workflowId: will be set after TG-132 is deployed (use PLACEHOLDER_TG132_ID)
   - Passes: review_id, author_name, rating, review_text, is_five_star

Connections:
- tg131-trigger -> tg131-fetch -> tg131-extract -> tg131-dedup -> tg131-save -> tg131-switch
- tg131-switch output 0 (negative) -> tg131-neg-alert -> tg131-call-drafter
- tg131-switch output 1 (5-star) -> tg131-call-drafter
- tg131-switch output 2 (4-star) -> tg131-call-drafter
