  <files>automation/n8n-workflows/TG-129-review-request-sequence.json</files>
  <action>
Create a new n8n workflow JSON for TG-129-review-request-sequence. This runs on a schedule, finds jobs completed ~24h ago, and sends review request SMS to those customers.

Why scheduled (not event-triggered): TG-128 already fires on job_completed. TG-129 runs as a scheduled batch to catch ALL completions and dedup against TG-128 positive-reply review requests.

Workflow flow:

1. Schedule Trigger (id: tg129-trigger, name: "Daily 10 AM Review Request Check")
   - type: n8n-nodes-base.scheduleTrigger, typeVersion: 1.2
   - rule: every day at 10:00 AM CT (15:00 UTC)

2. Fetch Recent Completions (id: tg129-fetch, name: "Fetch Jobs Completed 20-28h Ago")
   - type: n8n-nodes-base.httpRequest, typeVersion: 4.2
   - GET https://lwtmvzhwekgdxkaisfra.supabase.co/rest/v1/jobber_email_events?event_type=eq.job_completed with time window 20-28h ago using n8n expressions
   - Select: id,customer_name,customer_phone,customer_email,service_type,created_at
   - Headers: apikey and Authorization with SUPABASE_ANON_KEY_PLACEHOLDER
   - continueOnFail: true

3. Filter Has Phone (id: tg129-filter-phone, name: "Filter: Has Phone Number")
   - type: n8n-nodes-base.filter, typeVersion: 2
   - Condition: json.customer_phone is not empty

4. Check Already Requested (id: tg129-dedup, name: "Check If Review Already Requested")
   - type: n8n-nodes-base.code, typeVersion: 2
   - For each item, HTTP GET review_requests table by customer_phone in last 30 days
   - If no rows: include item. If rows exist: skip. On error: include anyway.
   - Use this.helpers.httpRequest() with Supabase REST API

5. Build SMS Payload (id: tg129-build-sms, name: "Build Review Request SMS")
   - type: n8n-nodes-base.code, typeVersion: 2
   - Message: "Hi {name}! Thank you for choosing TotalGuard Yard Care. We would love to hear how we did! Would you take 30 seconds to leave us a Google review? It means the world to our small business: https://g.page/r/CQEMNig5DhIBEAI/review - Thank you! The TotalGuard Team"
   - Output flat JSON: to_phone, message_body, workflow_name (TG-129-review-request), message_type (review_request)

6. Send via TG-94 (id: tg129-send-sms, name: "Send Review Request SMS via TG-94")
   - type: n8n-nodes-base.executeWorkflow, typeVersion: 1
   - workflowId: "AprqI2DgQA8lehij" (TG-94)

7. Log Review Request (id: tg129-log, name: "Log Review Request to Supabase")
   - type: n8n-nodes-base.httpRequest, typeVersion: 4.2
   - POST https://lwtmvzhwekgdxkaisfra.supabase.co/rest/v1/review_requests
   - Body: customer_phone, customer_email, customer_name, request_type: "sms_24h", source_workflow: "TG-129"
   - continueOnFail: true

Connections: tg129-trigger -> tg129-fetch -> tg129-filter-phone -> tg129-dedup -> tg129-build-sms -> tg129-send-sms -> tg129-log

Settings: executionOrder: "v1"

Supabase table needed (document in SUMMARY, user must create manually):

  CREATE TABLE IF NOT EXISTS review_requests (
    id uuid DEFAULT gen_random_uuid() PRIMARY KEY,
    customer_phone text,
    customer_email text,
    customer_name text,
    request_type text NOT NULL,
    source_workflow text NOT NULL,
    google_review_received boolean DEFAULT false,
    followup_sent boolean DEFAULT false,
    followup_sent_at timestamptz,
    created_at timestamptz DEFAULT now()
  );
  CREATE INDEX idx_review_requests_phone ON review_requests(customer_phone);
  CREATE INDEX idx_review_requests_created ON review_requests(created_at);
