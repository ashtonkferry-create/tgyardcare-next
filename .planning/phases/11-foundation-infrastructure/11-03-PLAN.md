**Step 2: Create the workflow template JSON.**

Create `automation/n8n-workflows/TG-AUTOPOST-social-publisher.json` with this structure:

```
Name: TG-AUTOPOST-social-publisher
Trigger: Execute Workflow Trigger (called by content workflows like TG-99 blog publisher, future social content generators)

Input expected:
- post_text: string (the caption/text)
- image_url: string (optional, URL of image to attach)
- platforms: string[] (which platforms to post to, e.g. ["instagram", "linkedin", "pinterest"])
- post_type: string ("blog_share", "review_highlight", "seasonal_tip", "before_after")
- schedule_at: string (optional ISO datetime for scheduled posting)

Nodes:
1. Execute Workflow Trigger -- receives input from calling workflows
2. Code node "Validate Input" -- validates required fields, removes "facebook" from platforms array (Facebook is hacked), defaults platforms to ["instagram", "linkedin", "pinterest"] if not specified
3. Code node "Build API Payload" -- constructs the payload for Upload-Post/LATE API. Use placeholder URL: `AUTOPOST_API_URL` and placeholder key: `AUTOPOST_API_KEY`
4. HTTP Request node "Publish Post" -- POST to AUTOPOST_API_URL with Authorization header using AUTOPOST_API_KEY. Send the constructed payload as JSON body.
5. Code node "Log Result" -- returns success/failure summary with platform statuses

Settings: executionOrder v1, saveManualExecutions true, callerPolicy "any"
Tags: ["autopost", "social-media", "phase-17"]
```

Make it a proper n8n workflow JSON matching the format used by other TG-* workflows (see TG-113 for structure reference). Use proper n8n node type versions (executeWorkflowTrigger v1, code v2, httpRequest v4.2).
